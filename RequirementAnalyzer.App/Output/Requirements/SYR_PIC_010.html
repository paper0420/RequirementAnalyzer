SYR_PIC_010


  • 7.4.1.15 SYR_PIC_010: flash update application
  • 7.4.1.15.1 Description
  • 7.4.1.15.1.0-1 Brief description:
  • 7.4.1.15.1.0-1.0-1 The “flash update” of PIC is handled by “Flash Bootloader”. The “Flash bootloader” will be started on request of the “bootloader” in “system CPU”. The bootloader in “system cpu” will execute complete UDS-communication with diagnosis-tester and extract PIC-related data. By this the “flash bootloader” of PIC has very simple, atomic interface: - Delete application program - Write new data to application-program memory (physical addressing) - Verify written data - Restart PIC (-> new application program) This is STATELESS interface, all states are handled by “bootloader” in “system cpu”. In case of failure (verification failed), the “bootloader” in “system cpu” is in charge to retry.
  • 7.4.1.15.1.0-2 Preconditions:
  • 7.4.1.15.1.0-2.0-1 The device is in one of the following system states: PIC was requested to enter “flash bootloader” via UART
