  • 7.9.1.11 SYR_COA_070: COM Adapter Debouncing mechanism
  • 7.9.1.11.1 Description
  • 7.9.1.11.1.0-1 Brief description: This UC gives brief description of Debouncing mechanism. The error debouncing for CAN and Flexray buses are done for parameterized number of events. For LIN busses error (Bus error, Node error, frame error) debouncing is done for parameterised time. The internal qualifier is updated depending on the COM status (CRC, Timeout, Alive error). If a time out error occurs for "n" consecutive cycles then a time out error is reported by COM Adapter. The number of consecutive cycles is calibratable.
  • 7.9.1.11.1.0-2 Preconditions: RUNLEVEL>=30
  • 7.9.1.11.1.0-3 Trigger: COM Adapter receives Signal value from COM
  • 7.9.1.11.1.0-4 Input data: Valid COM signal
  • 7.9.1.11.1.0-5 Description of behaviour: Com Adapter debouncing mechanism is done depending on the error status as per the time specifications. For example, K_T_CANA_BUSOFF_DEF is 60ms and K_T_CANA_BUSOFF_OK is 50ms. The Defect monitoring counter is incremented during error presence and after counter reached maximum value the error is confirmed. If the error is not present after error detection then healing counter is decremented when counter is zero then Error debounced is reset to “0”.
  • 7.9.1.11.1.0-6 Timing Requirements: OK condition: OK debounce counter * cycle time of the scheduling table ms Defect condition: DEF debounce counter * cycle time of the scheduling table ms
  • 7.9.1.11.1.0-7 Output data: When input data is valid COM Adapter transfers COM signals with Qualifier information When input data is invalid COM Adapter transfers COM signals as invalidity value and records the DTC through DEM.
