  • 7.7.1.14.1.0-1 Brief description: This is only for Meta.2017 in I20 project. It is about moving XFF into Bootloader. 14.04.2021 G. Teiusan: SYR became obsolete, due to drop of XFF functionality. The XCP functionality is still valid and covered by SYR_SEC_140.
  • 7.7.1.14.1.0-2 Preconditions: CCU is operational
  • 7.7.1.14.1.0-3 Trigger: XCP Flashing Request
  • 7.7.1.14.1.0-4 Input data: none
  • 7.7.1.14.1.0-5 Description of behaviour: XCP Flashing Request shall be received in the Appl SW module The CCU shall only accept the XCP Flashing Request when: ECU Mode = Engineering Mode AND The SFA Token: "XCP Write" w/Feature ID: 0x001D1B, is installed AND valid. When the Pre-conditions of Point.2 are met, the Appl SW module shall: Set the regular XFF flag. Perform non destructive reset for flashing.
