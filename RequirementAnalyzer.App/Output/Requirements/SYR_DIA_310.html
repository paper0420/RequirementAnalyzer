SYR_DIA_310


  • 7.5.1.31 SYR_DIA_310: generate Reset DTC
  • 7.5.1.31.1 Description
  • 7.5.1.31.1.0-1 Brief description: A DTC (ID: 0xBA0023 = Sammelfehler Software) is generated when a spontanous reset occurs to the CPU. The DTC can only be entered into DEM AFTER the actual event occured during the next system startup. The actual DTC monitor is the RGM module of the CPU which captures the reset input from multiple possible input sources and provides this information in a corresponding hardware register. This information is part of the environmental data for this DTC. Other environmental data has to be sampled either continously or event driven in a RAM area that is not lost with system restart. A more detailed description is available here: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Diag/bmw_ccu_sammelfehler_sw.pptx DTC details and environmental data is described here: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Diag/sammelfehler_sw_i400.xlsx Environmental data has to be retrieved from hardware registers (HWREG)
  • 7.5.1.31.1.0-2 Preconditions: none - software runs or CPU is in sleep state
  • 7.5.1.31.1.0-3 Trigger: various input reset sources as described in associated document
  • 7.5.1.31.1.0-4 Input data: DTC monitor output hardware reset register of CPU and collected environmental data
  • 7.5.1.31.1.0-5 Description of behaviour: DTC is announced to DEM and collected environmental data is provided upon request from DEM.
  • 7.5.1.31.1.0-6 Timing Requirements: none
  • 7.5.1.31.1.0-7 Output data: DEM entry and Environmental Data
  • 7.5.1.31.1.0-8 Postconditions: none.
  • 7.5.1.31.1.0-9 Descriptions of exceptions: none
  • 7.5.1.31.1.0-10 Dependencies and interactions: DTC and freeze frame is shared with FuSa. See associated specifications for details.
