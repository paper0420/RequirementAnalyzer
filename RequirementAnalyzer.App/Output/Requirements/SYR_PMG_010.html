SYR_PMG_010


  • 7.2.1.1 SYR_PMG_010: cancel shutdown if device is needed again
  • 7.2.1.1.1 Description
  • 7.2.1.1.1.0-1 Brief description:
  • 7.2.1.1.1.0-2 This UC is strong remainder to consider that at any time a wake up event can occur, even if the CCU is processing shutdown-procedure. By timing constraints it is not always possible to finish the complete shutdown procedure and restart from beginning. In these cases the shutdown procedure has to be aborted in a safe / consistent manner to prevent any kind of inconsistency (RAM/NVM) which could lead to unwanted behaviour.
  • 7.2.1.1.1.0-3 Preconditions: device shutdown is ongoing
  • 7.2.1.1.1.0-4 Trigger: any event which could cause wakeup of CCU
  • 7.2.1.1.1.0-5 Input data: wake up signal
  • 7.2.1.1.1.0-6 Description of behaviour: the ongoing shut down is canceled and CCU becomes operative again
  • 7.2.1.1.1.0-7 Timing Requirements: within 150ms (same like wakeup from sleep) CCU has to be operatable again
  • 7.2.1.1.1.0-8 Output Data: none
  • 7.2.1.1.1.0-9 Postconditions: CCU operates normally
  • 7.2.1.1.1.0-10 Descriptions of exceptions: none
  • 7.2.1.1.1.0-11 Dependencies and interactions: none
  • 7.2.1.1.2 Differences to CRS: none
  • 7.2.1.1.3 Questions and Answers: none
