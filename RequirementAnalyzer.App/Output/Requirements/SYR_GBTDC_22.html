  • 1.1.29.11 SYR_GBTDC_22: Easy Charge China - long VIN to be sent
  • 1.1.29.11.1 Description
  • 1.1.29.11.1.0-1 Brief description: According to the GB/T-communication protocol, the optional VIN can be sent from the vehicle to the charging station on the DC_CAN bus.
  • 1.1.29.11.1.0-2 Preconditions: CCU is able to sense proximity
  • 1.1.29.11.1.0-3 Trigger: Plug is inserted in "DC Ladedose" intelligent socket
  • 1.1.29.11.1.0-4 Input data: Coding Data (NVM)
  • 1.1.29.11.1.0-5 Description of behaviour: Support Easy Charge China with new concept by sending the 17 Bytes of long VIN on the DC_CAN. Currently the BRM frame is already sent on the bus with 49 Bytes (over TP), the optional BN Signals (such as VIN) are filled with 0xFF. The VIN is re-defined in the dbc and shall be filled in before sending it on the bus. DBC and BN Mapping -> Integrate new dbc from BMW. Previously, VIN was 1 signal of 17 bytes. Now VIN is split to 17 signals with 1 byte of data each. -> Integrate arxml i.e. signal/interface mapping (17 bytes = 17 signals) with CIF component Coding -> Integration of coding parameter(longVin) utlizing updated coding arxml and CAFD -> Interface from CIF module (Sender-Receiver port) to coding wrapper
  • 1.1.29.11.1.0-6 Timing Requirements: NA
  • 1.1.29.11.1.0-7 Output data: NA
  • 1.1.29.11.1.0-8 Postconditions: Verification is required if 17 bytes VIN is sent on bus.
  • 1.1.29.11.1.0-9 Descriptions of exceptions: NA
  • 1.1.29.11.1.0-10 Dependencies and interactions: CIF component, ComAdapter, Com, RTE, Coding Wrapper, BAC
