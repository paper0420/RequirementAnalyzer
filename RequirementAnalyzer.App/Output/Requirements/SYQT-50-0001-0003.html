#SYQT-50-0001-0003#

[Charging],[Component_Protection],[Over_Temperature]
Test Objective: Overtemperature detection confirmation (3-phase)

Requirements

  • 18429 -
    [Not Changed][accepted][SYQT (ENG.10)SYIT (ENG.9)][Requirement]
    The charger shall stop operation when the maximum temperature limit is exceeded.
  • 18430 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    If an overtemperature is detected, the funciton "Komponentenschutz und Derating" shall set the "Abschaltung aufgrund Übertemperatur" bit in the error status word.
  • 18431 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The excess temperature diagnostics shall be performed in every operating state.
  • 11425 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    Pre-Condition: a) activate and operate the development object at input voltage (± 15% of the rated input voltage) and input frequency (50 Hz ± 2%).
  • 17634 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    Action: a) raise the temperature of the temperature monitoring point of the development object from room temperature to exceed the set value specified by BMW.
  • 11426 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    Post-Condition: a) development object must automatically enter into overtemperature protection state. b) development object must run with reduced power or shut down. c) development object must be able to restore automatically when the temperature restores to normal.
