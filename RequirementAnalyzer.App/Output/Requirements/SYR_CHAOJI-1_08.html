  • 1.1.32.8 SYR_CHAOJI-1_08 : provide functionality in SW to handle message transmission and reception during Version Negotiation
  • 1.1.32.8.1 Description
  • 1.1.32.8.1.0-1 Brief description: This SYR covers the handling of TX-RX messages during execution of "Version Negotiation" function module in Chaoji-sequence.
  • 1.1.32.8.1.0-2 Preconditions: Vehicle is coded as Chaoji vehicle (VehChrTypDC = 4). BMW_nr_DcCanProtReq is set to 2 (Chaoji vehicle charges at chaoji-1 charger).
  • 1.1.32.8.1.0-3 Trigger: In TX, data provided by LIM-C via record. In RX, data received at CCU from charger
  • 1.1.32.8.1.0-4 Input data: In TX, SW record (all TX-signals) is written by LIM-C.   • 1.1.32.8.1.0-5 Description of behaviour: During Chaoji-1 charging sequence, in "Version Negotiation" function module, Charger sends "Charger Protocol Version" message to CCU. Vehicle (CCU) sends "Vehicle Negotiation Results" message to Charger. Function Module Contents of Parameter Group Message type Message content (short) Version Negotiation Charger Protocol Version PV CPV Vehicle Negotiation Results PV VPV In TX phase (CCU to Charger), LIM-C shall provide the (write TX-signals via record) message-data to COM Adapter via RTE and similarly. COM receives the data from COM Adapter. Based on configuration at COM, COM transmits the data to lower layer and on DC-CAN. In RX phase (charger to CCU), message is received by vehicle (CCU), RX-PDU is sent from CAN IF to upper layers (COM). RX-signals are received by LIM-C from ComAdapter via RTE interface.   • 1.1.32.8.1.0-6 Timing Requirements: NA
  • 1.1.32.8.1.0-7 Output data: "Vehicle Acknowledgement Results message" shall be seen on bus when in transmission. RTE interface update shall done when "Charger Protocol Version" message is received at CCU.
  • 1.1.32.8.1.0-8 Postconditions: NA
  • 1.1.32.8.1.0-9 Descriptions of exceptions: Send Trigger for "Version Negotiation" is not in scope.
  • 1.1.32.8.1.0-10 Dependencies and interactions: Supplier-SW (Chaoji-Adaptor), RTE, COM, Supplier-SW (Chaoji-TP), PduR, BSWM
