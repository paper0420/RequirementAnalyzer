  • 7.7.1.19.1 Description
  • 7.7.1.19.1.0-1 Brief description: Mechanism to allow update of HSM firmware.
  • 7.7.1.19.1.0-2 Preconditions: old HSM version flashed

  • 7.7.1.19.1.0-4 Input data: PDx with HSM firmware
  • 7.7.1.19.1.0-5 Description of behaviour: Brief description: The HSM update shall have no logistic to be reported in DSC: SVK data. The HSM update shall occur independently with exclusive access to the flash memory. The Main controller shall update the HSM SW using the following CSAIs and sequence as in the graph "HSM Update" in Catalog. The CCU shall execute a reset at the end of an HSM update sequence. The HSM's SWFK logistics shall follow the guideline: swfk_00009a09_(HSM Version). The System shall allow the activation of the HSM's Exclusive Flash Access Setting via Supplier RID Once the HSM's Exclusive Flash Access Setting is enabled, the System shall allow the HSM update only via the special(with defined logistic) PDX reserved for this purpose The RID to activate the HSM's Exclusive Flash Access Setting is defined in the "Catalog" sheet In case of HSM update failure (HSM in BootMode), following the system reset, the system shall enter BTLD mode and allow HSM re-flashing (as detailed in "Boot Sequence" and "Overall HSM Flashing Sequence" in "Catalog" sheet, and in relation with "CCU-26855 HSM rollback prevent" epic) Please refer to the sheet "catalog" in the file https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Documentation/Security/SysRequirementsForSecurityEpics.xlsx
