#SYQT-68-0001-0005#

[Charging],[DC_Charging],[Proximity]
Test Objective: Confirm that analog voltage notification of DC proximity terminal(CC2) is performed correctly.(CHAdeMO)

Requirements

  • 19065 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    Every 10 ms, the supplier function shall send the state of the connector proximity detection line (A_T_PROXI) to the "Ladeschnittstelle" function via the "Status_Connector_Proximity_Detection" interface signal (0 = inactive, 1 = active, 2 = error)
  • 19067 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    The "Ladeschnittstelle" function notifies the supplier function via the "Ansteuerung_Connector_Proximity_Detection_Transistor_N" signal in a 10 ms time grid of the requested actuation of the transistor-N of the connector proximity detection according to the CHAdeMO specification.
  • 19075 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    The "Ladeschnittstelle" function notifies the supplier function via the "Vehicle charge permission" signal in a 10 ms time grid that the charge permission line ("A_T_Char_permission") shall be activated or deactivated (0 = deactivate, 1 = activate).
  • 24904 -
    [Not Changed][partly accepted][SYQT (ENG.10)][Requirement]
    The supplier function shall measure the voltage at the input CC2 and send the value to the "Ladeschnittstelle" function.
