  • 7.11.2.10 SYR_SF10 FuSa Implementation - Flash Boot Loader Safety
  • 7.11.2.10.1 Description
  • 7.11.2.10.1.0-1 Brief description: The boot loader starts executing after reset. Permanent and transient faults affecting the boot ROM can leads to wrong execution of the application software at the end of the boot procedure. Such alteration to be detected by a strong control flow mechanism linked to a system Boot monitoring Safe boot shall ensure: 1. Protection against manipulation for ECUs with safety relevent vehicle data 2. Safeguarding of data integrity in the flash memory 3. Protection against manipulation of safety relevent output and input Pin and BUS-Communication. Applications must use SWT to monitor boot time errors before initiating safety tasks. If the application detects a boot-time error, the safety function shall not start and the application shall send a failure indication to the system. Flow of control sate to be monitoted using WDG Bootloader in a protected memory area of the ECU CCU shall maintain safe state if the number of consecutive resets exceeds the specified number. Bootloar shall not change the safety relevent output and input Pin and BUS-Communication. Bootloar shall not change the safety relevent configuration values Safe state: Safety function shall not start and the safety application/stay in reset state
