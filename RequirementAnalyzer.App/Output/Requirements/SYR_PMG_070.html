SYR_PMG_070


  • 7.2.1.6 SYR_PMG_070: detect and handle wakup request
  • 7.2.1.6.1 Description
  • 7.2.1.6.1.0-1 Brief description:
  • 7.2.1.6.1.0-2 This UC gathers all common tasks needed to make CCU operable caused by any start reason. This contains the start of hard- and software. This UC terminates if the first command is executed by CPU (next steps are taken by SYR_PMG210) This UC is responsible to determine the source of wakeup and forward to software.
  • 7.2.1.6.1.0-3 Preconditions: CCU is in minimal power consumtion mode
  • 7.2.1.6.1.0-4 Trigger: wake up request indication by dedciated interface (CAN/Charge Plug/internal RTC)
  • 7.2.1.6.1.0-5 Input data: wake up event
  • 7.2.1.6.1.0-6 Description of behaviour: CCU will wake up and starts operating
  • 7.2.1.6.1.0-7 Timing Requirements: <150ms after wake up event, the CCU must be able to process CAN messages
  • 7.2.1.6.1.0-8 Output data: none
