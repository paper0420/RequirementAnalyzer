  • 2.1.34 SYR_ACC400: Change of charging duration calculation
  • 2.1.34.1 Description
  • 2.1.34.1.0-1 Brief description: Separate charging counter should be implemented for single charging session, and it should be mapped to current existing freeze frame(0x710A).
  • 2.1.34.1.0-2 Preconditions: Always
  • 2.1.34.1.0-3 Trigger:
  • 2.1.34.1.0-4 Input data:
  • 2.1.34.1.0-5 Description of behaviour: In addition to existing charging lifetime counter of CCU, an additional charging session counter shall be implemented and it should be connected to the existing freezeframe (0x710A). The existing freeze frame is already connected to lifetime counter, so in this implementation, it shall be disconnected from lifetime counter and shall be connected to this newly calculated charging session counter. This charging session counter starts when any of the chargermodules are in charge state((Tar_OPM == 2) and it shall reset if all the charger modules are went into standby state((Tar_OPM == 1) . Counter increment happens for every 2 sec.
  • 2.1.34.1.0-6 Timing Requirements: Counter service routine is 2 sec
  • 2.1.34.1.0-7 Output data: Charging session counter
