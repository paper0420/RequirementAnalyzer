SYR_PMG_060


  • 7.2.1.5 SYR_PMG_060: detect and handle wakup via timer
  • 7.2.1.5.1 Description
  • 7.2.1.5.1.0-1 Brief description:
  • 7.2.1.5.1.0-2 The CCU needs to wake up after certain time has passed. While the time until wakeup shall happen passes, the CC will consume minimal power. After time has passed, the CCU will wake up and execute planned tasks. This will be done by external ECU: On demand the CCU will send request to external ECU to set alarm-time. If alarm-time expired, this external ECU will wakeup the ECAN (via wakeup-line) and inform CCU about expired time.
  • 7.2.1.5.1.0-3 Preconditions: EVSE connected, V2G communication results in "charge pause", CCU has to enter
  • 7.2.1.5.1.0-4 Trigger: request to execute "charge pause"
  • 7.2.1.5.1.0-5 Input data: pause time
  • 7.2.1.5.1.0-6 Description of behaviour: CCU will enter minimal power consumption until pause time has been elapsed
  • 7.2.1.5.1.0-7 Timing Requirements: depens on V2G communication result (charge schedule calculation)
  • 7.2.1.5.1.0-8 Output data: none
  • 7.2.1.5.1.0-9 Postconditions: charging is continued
