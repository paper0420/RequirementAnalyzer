  • 4.2.1.7 SYR_DRF80: Synchronism between the pedal sensors measurement values
  • 4.2.1.7.1 Description
  • 4.2.1.7.1.0-1 Brief description: Synchronizing the measurement values coming from the driving pedal sensors
  • 4.2.1.7.1.0-2 Preconditions:KL15 ON
  • 4.2.1.7.1.0-3 Trigger:
  • 4.2.1.7.1.0-4 Input data: - Sensor 1 signal (FP_1_Meas_MCU signal --> ACCEL1) - Sensor 2 signal (FP_2_Meas_MCU signal --> ACCEL2)
  • 4.2.1.7.1.0-5 Description of behaviour: the two sensor measurement input signals have to be synchronized before performing the necessary plausibilisation, which implies that FP_2_Meas_MCU=0,5*FP_1_Meas_MCU. The information related to the sampling time of the corresponding values on the corresponding AD-Converter should be available in the SW. Based on the sampling time of the values of Sensor 1 and Sensor 2, the plausibilisation should be performed only between sensors values which have been sampled at the same time +/- a certain tolerance (TBD).
