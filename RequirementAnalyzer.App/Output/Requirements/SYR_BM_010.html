  • 7.4.1.2 SYR_BM_010: select application to execute
  • 7.4.1.2.1 Description
  • 7.4.1.2.1.1 Brief description:
  • 7.4.1.2.1.1.0-1 The bootmanager is first application which will be executed after reset of main-cpu was executed. The bootmanager will decide which application will be started.
  • 7.4.1.2.1.1.0-2 This decision is based on: Validity of application: The Bootmanger will check if main application/bootloader is valid. If the affected program code is “corrupted”, the bootmanager will NEVER start this code. Request of application: The main application will request “bootloader” if sw-update of main-application is requested via diagnosis. The bootloader will request “ main-application” if sw-update of main-application has passed The Bootmanager will NEVER terminate any application (this is task of application itself!). The Bootmanager will execute all SAFE-Boot related functions. The bootloader-update application will be handled like “main application”, since it is a replace of “main application”
  • 7.4.1.2.1.1-1 Preconditions:
  • 7.4.1.2.1.1-1.0-1 The device is in one of the following system states: SOC is powered and reset was executed

  • 7.4.1.2.1.1-2.0-1 Reset of system cpu
