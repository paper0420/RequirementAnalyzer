  • 7.4.2.13 SYR_SUP_120: Operate minimal power management
  • 7.4.2.13.1 Description
  • 7.4.2.13.1.0-1 Brief description:
  • 7.4.2.13.1.0-2 This use-case will implement minimal power-management to : - Prevent endless operation (=power consumption) of CCU is anything failed while software update - Prevent sleep-mode of CCU and/or car while software update is ongoing - Enable retries to flash-update the CCU under any condition (This will include re-configuration/reset of communication/software-update layer of CCU to have a “virgin” start-up-condition for next try. - Enable readout of any data for “failure reason” by diagnosis-tester This will be done by: - Minimal participation in Network-management - enabling to enter sleep-mode (low power consumption) if flash-update failed and certain time has passed - enabling of wakeup the CCU in case of external applied wakeup request (via CAN/Ethernet) is detected - complete initialisation of communication/software-update layer if retry is started - Detection connection to diagnosis tester and keep car-networks running as demanded
  • 7.4.2.13.1.0-3 Preconditions: sw update ongoing
  • 7.4.2.13.1.0-4 Trigger: start of sw-update
  • 7.4.2.13.1.0-5 Input data: none
  • 7.4.2.13.1.0-6 Description of behaviour: CCU will reduce all its functionality to "sw-update-needed" only
