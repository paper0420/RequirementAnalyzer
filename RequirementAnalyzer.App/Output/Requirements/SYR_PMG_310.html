  • 7.2.1.28 SYR_PMG_310: Power State Handling on private CAN (ILD)
  • 7.2.1.28.1 Description
  • 7.2.1.28.1.0-1 Brief description:There is some conceptual problem while shutdown the private CAN: If the shutdown timing of private CAN is "not good", there can be a problem of SYNC loss between CCU and ILD. This should be solved by change of ILD shutdown activities like described in https://asc.bmw.com/jira/browse/CCU-27337

  • 7.2.1.28.1.0-2 Preconditions: CCU decides to shutdown the private CAN
  • 7.2.1.28.1.0-3 Trigger: no trigger
  • 7.2.1.28.1.0-4 Input data: ILD State Machine which is followed, https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Documentation/VehicleIF/CAN/iLD_Mode_Management/CCU-27337
  • 7.2.1.28.1.0-5 Description of behaviour: CCU will extrapolate state of ILD and try to prevent SYNC-LOSS
  • 7.2.1.28.1.0-6 Timing Requirements: timing is same like requested in KLH, only additional information need to be considered
  • 7.2.1.28.1.0-7 Output data: none (private CAN enteres stable sleep without invalid signal qualification)
  • 7.2.1.28.1.0-8 Postconditions: none (this case will never terminate)
  • 7.2.1.28.1.0-9 Descriptions of exceptions: none
  • 7.2.1.28.1.0-10 Dependencies and interactions: none
