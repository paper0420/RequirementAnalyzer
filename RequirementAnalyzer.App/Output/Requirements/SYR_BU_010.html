  • 7.4.1.3 SYR_BU_010: update bootloader-application
  • 7.4.1.3.1 Description
  • 7.4.1.3.1.1 Brief description:
  • 7.4.1.3.1.1.0-1 The bootloader-updater will override the main application. The binary of the bootloader-update contains the binary image of the new bootloader. After CCU reset, the bootloader-update will delete the old bootloader and copy the new bootloader to assigned flash-memory. If update was successful, the bootloader-updater will delete its own signature (in flash memory) and executes a reset of CCU. The CCU will restart, since the main application is missing, the new bootloader will be started and new main Application has to be flashed by external device. The bootloader-updater will store logging data in “config data” memory for tracing while flash-process is ongoing. (This is independent of any DLT-tracing)
  • 7.4.1.3.1.1.0-2 Preconditions:
  • 7.4.1.3.1.1.0-3 The device is in one of the following system states: SOC is powered and main Application was replaced by “bootloader updater” application.
  • 7.4.1.3.1.1-1 Trigger:
  • 7.4.1.3.1.1-1.0-1 Replace main Application by Bootloaderupdater-application
