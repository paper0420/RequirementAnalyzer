  • 1.1.30.15 SYR_PLC_13: provide functionality to trigger PLC Secondary DTCs in case of SLAC error or V2G stack error
  • 1.1.30.15.1 Description
  • 1.1.30.15.1.0-1 Brief description: Secondary DTCs information is present in ZEDIS and must be implemented.
  • 1.1.30.15.1.0-2 Preconditions: Refer to ZEDIS document
  • 1.1.30.15.1.0-3 Trigger: Refer to ZEDIS document
  • 1.1.30.15.1.0-4 Input data: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Diag/Zedis_Export/
  • 1.1.30.15.1.0-5 Description of behaviour: InI20/G26, BMW decided to implement Secondary DTCs for PLC SLAC failure and V2G Communication failure instead of primary DTCs from G08. Primary DTCs (from G08 -> shall not be applicable now in I20/G26): DTC 0x03194B PLC-Com-Error-> DTC is set if SLAC attempt has failed DTC 0x03194C PLC Protocol Error-> DTC is set if a protocol error is reported by SCC. New Secondary DTCs (defined in ZEDISv24 -> shall be applicable in I20/G26) 0x031BB8 - Power Line Communication: Protokollfehler 0x031BB9 - Power Line Communication: Verbindungsstörung, Ladesäule DTC Description UWBs* 0x03194A Power line communication: communication to modem disturbed No PLC specific Freeze Frame to be implemented 0x031BB8 Power line communication: protocol fault 0x1768, 0x1769, 0x4020, 0x4021, 0x4022, 0x4023, 0x70AB 0x031BB9 Power line communication: connection fault, charging station 0x1768, 0x1769, 0x4024 *Freeze Frame (UWBs) implementation is covered with SYR_PLC_12. This is not in scope of this SYR. In I20/G26 SW, the logic (set or clear conditions for primary DTCs from G08) shall now connect to secondary DTCs. DEM/NVM configuration shall be checked or modified to correctly set new error entry. DTCs are expected to be set whenever SW reports PLC SLAC error (DLINK not established due to SLAC process failure in between) or whenever V2G SCC error (message timeouts, negative response code, transport layer errors etc.)
  • 1.1.30.15.1.0-6 Timing Requirements: Whenever SLAC error or V2G SCC error is reported, DTC shall be reported in fault memory.
  • 1.1.30.15.1.0-7 Output data: DTCs and respective UWBs (in scope of SYR_PLC_12) shall be captured to understand the fail case.
  • 1.1.30.15.1.0-8 Postconditions: PANA Appl reports to BMW SWC on Slac Error or V2G error through interfaces. Control of state machine is by BMW SWC.
