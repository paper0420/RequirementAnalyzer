SYR_DIA_520


  • 7.5.1.53 SYR_DIA_520: PMD Data read out via diag job
  • 7.5.1.53.1 Description
  • 7.5.1.53.1.0-1 Brief description: Diagnostic jobs needed to read out the PMD (post-mortem-dump) data from RAM and from NVM (flash). These are introduced in Zedis V24.
  • 7.5.1.53.1.0-2 Preconditions: CCU in engineering or plant mode. Diagnostic default or extended session started
  • 7.5.1.53.1.0-3 Trigger: run diag job 0xAC24 (PMD_Read_From_RAM) or 0xAC25 (PMD_Read_From_Flash)
  • 7.5.1.53.1.0-4 Input data: NA
  • 7.5.1.53.1.0-5 Description of behaviour: - When diag job 0xAC24 is run, PMD data from RAM is retrieved. - When diag job 0xAC25 is run, PMD data from NVM (flash) is retrieved. The diagnostic jobs read out up to 4kB data section in RAM or program flash. Content of PMD is dynamic and depends on kind of reset. The 4kB data is not secured and contains only information retrieved in Application (Bootloader is excluded): this is as per Zedis V24.
  • 7.5.1.53.1.0-6 Timing Requirements: NA
  • 7.5.1.53.1.0-7 Output data: PMD data from Application
