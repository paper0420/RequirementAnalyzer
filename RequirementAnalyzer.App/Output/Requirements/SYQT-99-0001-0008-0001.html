#SYQT-99-0001-0008-0001#

[Functional_Safety],[Functional_Safety],[Functional_Safety]
Test Objective: HVAC input current is limited to limit value (3 phase) Charging with left turning grid *) Details Confirm AC charge disable when HVAC voltage drop down. Inform BMW SW

Requirements

  • 23681 -
    [Not Changed][partly accepted][SYQT (ENG.10)SYIT (ENG.9)][Requirement]
    While AC charging, the CCU shall limit the input current on HVAC (HVAC_I_L1Sig, HVAC_I_L2Sig, HVAC_I_L3Sig) as follows: - 18A per phase for 3.5KW variant (1 phase) - 36A per phase for 7KW variant (1 phase) - 18A per phase for 11KW variant (3 phase) - 54A per phase for 11KW variant (1 phase) - 73A per phase for 15KW variant (1 phase) - 36A per phase for 22KW variant (3 phase) FTT: 10000ms Safe state: Disable the AC charging and inform BMW SW
  • 18395 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The "Komponentenschutz und Derating" function shall stop operation of the charging electronics when a protection condition is present.
  • 18397 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The "Komponentenschutz und Derating" function shall trigger an operating mode change to the "Error" state when a protection condition is present.
  • 18399 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    If a self-protection emergency deactivation takes place, an entry shall be made in the error memory clearly identifying the cause of the error.
  • 18418 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    If an AC overcurrent is detected, the function "Komponentenschutz und Derating" shall set the "Abschaltung Maximalstrom AC" bit in the error status word.
  • 18419 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    These diagnostics shall be performed only if the "Charge" target operating mode (2) is commanded by the "LDK" function on the respective charging module.
