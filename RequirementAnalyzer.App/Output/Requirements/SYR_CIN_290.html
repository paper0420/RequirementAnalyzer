  • 8.1.33 SYR_CIN_290: Sensor_Ident_Lesen for DV2-LIN
  • 8.1.33.1 Description
  • 8.1.33.1.0-1 Brief description: Implementation of the "Sensor_Ident_Lesen" for DV2 sensor data.
  • 8.1.33.1.0-2 Preconditions: CCU is running and the LIN Communication is correctly working.
  • 8.1.33.1.0-3 Trigger: After LIN wake-up, each diag request will be executed once for DV2 slave.
  • 8.1.33.1.0-4 Input data: Refer: https://asc.bmw.com/jira/secure/attachment/1673533/LIN_Meta_Data_Requests_v1.xlsx Value of STAT_SENSOR_VERBAUORT_DATA is 0x3F90
  • 8.1.33.1.0-5 Description of behaviour: DID for DV2 diag job is 0x22 16 09 LIN master sends multiple diag request and read-out response from slave. The response from slave for each request is used as a response parameter for DV2 Sensor_Ident_Lesen job. Diag Requests by LIN Master (on LIN bus for DV2): 1. BMW Teilenummer (BMW part number) 2. LIN Identtification data 3. Produktionsdatum (Production Date) 4. Seriennummer (Serial Number) 5. Softwarestand des Sensors (reference SW number) 5. Hardwarestand des Sensors (reference HW number) Response Parameter (24 Bytes) (prepared from DV2 slave response) for Sensor_Ident_Lesen job: 2 Bytes : Verbauort (provided by BMW) 6 Bytes : BMW Part Number 2 Bytes : Supplier Number 3 Bytes : Part Number (optional, 00 00 00) 3 Bytes : Production Date 4 Bytes : Serial Number 2 Bytes : SW Release Number 2 Bytes : HW Release Number
  • 8.1.33.1.0-6 Timing Requirements: none
  • 8.1.33.1.0-7 Output data: none
  • 8.1.33.1.0-8 Postconditions: none
  • 8.1.33.1.0-9 Descriptions of exceptions: none
  • 8.1.33.1.0-10 Dependencies and interactions: Autosar stack which are involved in the LIN\CAN Communication

  • 8.1.34 SYR_CIN_300: provide functionality to enable Ethernet Bus and Grauitous ARP in Production and Smart Charging - Obsolete (Refer SYR_CIN_601)
  • 8.1.34.1 Description
  • 8.1.34.1.0-1 Brief description: ETH bus (link-up/port config) are required to be done for Production (Running ETH diagnostic jobs) and Smart Charging (send Charging Profile to MGU). To control the start-up and shutdown of ETH, BMW has provided new interface. Also, it is required to seperately control the start-up and shutdown of SOME/IP SD using existing BMW interface. Existing interface shall not control ETH activation.
  • 8.1.34.1.0-2 Preconditions: CCU is running
  • 8.1.34.1.0-3 Trigger: BMW interface shall bring-up or down the ETH or SOME/IP services.
  • 8.1.34.1.0-4 Input data: NA
  • 8.1.34.1.0-5 Description of behaviour: Enable (start-up) or disable (shutdown) the ETH bus shall be done based on BMW_b_NetComAcvn interface value. Gratuitous ARP message shall be sent out once when BMW_b_NetComAcvn is enabled and ETH link is up. SOME/IP SD servies shall be set to enable (active)or disable (shutdown) based on BMW_b_SomeIPNetProtOnOff interface. It is the responsibility of BMW to control both the interfaces depending of use-case (Production and Smart Charging). Seperation of interfaces in Smart Charging (SOME/IP) sequence flow is depicted in below file: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/Sequence/LH_SOME_IP_CCU_13462_CCU_16475.pdf Calibratable Switch are provided as below BMWhvcha_s_b_NetComAcvn_C BMWhvcha_s_b_IPNetProtOnOff_C
  • 8.1.34.1.0-6 Timing Requirements: NA
  • 8.1.34.1.0-7 Output data: NA
  • 8.1.34.1.0-8 Postconditions: ETH bus shall be active (Link is up, Port is configured), Grauitous ARP message is present on bus and SOME/IP SD messages are seen on the bus (in Smart Charging only)
