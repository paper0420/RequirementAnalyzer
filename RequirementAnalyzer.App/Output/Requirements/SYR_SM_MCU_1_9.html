  • 7.10.1.1.153.1.0-1 Brief description:The safety goal is that the device does not leave its safe states for intervals equal or longer than the FTTI unless configured by the application software to do so. Safety analysis considers different states as Safe: 1) Unpowered is a safe state 2) Reset is a safe state 3) Correct operation, defined as all safety relevant modules operating according to specifications, is a safe state 4) Indicating an internal error is a safe state.
