SYR_DIA_340


  • 7.5.1.35 SYR_DIA_340: measure and read Stack Usage
  • 7.5.1.35.1 Description
  • 7.5.1.35.1.0-1 Brief description: The maximum usage of stack memory is monitored per task/ISR as a percentage value. The largest value over all monitoring values shall be logged into NVM. Only values larger than the last logged value shall be stored. See EPIC description for further details.
  • 7.5.1.35.1.0-2 Preconditions: Software runs.
  • 7.5.1.35.1.0-3 Trigger: A new value is generated once per second.
  • 7.5.1.35.1.0-4 Input data: Change of pre-allocated pattern in stack memory.
  • 7.5.1.35.1.0-5 Description of behaviour: This is a cyclic task.
  • 7.5.1.35.1.0-6 Timing Requirements: This is a low priority task executed every second.
  • 7.5.1.35.1.0-7 Output data: Current maximum stack usage value per Core.
  • 7.5.1.35.1.0-8 Postconditions: NVM remains over reset and is erased when the CCU is flashed.
  • 7.5.1.35.1.0-9 Descriptions of exceptions: none
  • 7.5.1.35.1.0-10 Dependencies and interactions: none
