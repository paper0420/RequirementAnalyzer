  • 4.1.1.1 SYR_CRA17: Generation of the qualifier for the Kl30C (Kl30C_Sts_qi)
  • 4.1.1.1.1 Description
  • 4.1.1.1.1.0-1 Brief description: In order to monitor the signal quality of the voltage level of the Kl30C, it has to be generated the corresponding qualifier of the signal, which consists in a uint value between 0 and 15. The qualifier value will be set accordingly with criteria of - availability; - unavailability; - unplausibility; - detection of errors at the HW-interface which have been defined by BMW (10224609_Funktionsschnittstellen_V03.pdf).
  • 4.1.1.1.1.0-2 Preconditions:
  • 4.1.1.1.1.0-3 Trigger:Kl15-ON
  • 4.1.1.1.1.0-4 Input data: Voltage signal of the Kl30C
  • 4.1.1.1.1.0-5 Description of behaviour: For the incoming I-Step (I300) only the qualifier values 0 (valid signal) and 15 (invalid signal with confirmed presence of a HW-failure) will be generated. These are the criteria for the signal qualifier generation for I-300: - Kl30C_Sts_qi=0 if ( 9V< Kl30C_Sts< 16V AND CAN_Crash=='No Crash detected') OR (0V < Kl30C_Sts<3V AND CAN_Crash=='Crash detected'); - Kl30C_Sts_qi=15 if (Kl30C_Sts > 16V) OR (3V < Kl30C_Sts < 9V) OR (9V< Kl30C_Sts< 16V AND CAN_Crash=='Crash detected') OR (0V < Kl30C_Sts<3V AND CAN_Crash=='No Crash detected') Note: CAN_Crash signal TBD
  • 4.1.1.1.1.0-6 Timing Requirements:
  • 4.1.1.1.1.0-7 Output data:Kl30C_Sts_qi, i.e. the signal qualifier of the voltage signal Kl30C_Sts
  • 4.1.1.1.1.0-8 Postconditions:
  • 4.1.1.1.1.0-9 Descriptions of exceptions: in case the Kl30C signal is not present at the corresponding PIN the CCU : - the KL30C_Sts Qualifier will be set to invalid
  • 4.1.1.1.1.0-10 Dependencies and interactions: As clearly indicated in the section 'Description of behaviour', there is a dependency with the value of CAN_Crash; this dependency should not be anymore considered as soon as a proper HW-failure detection (ShortToGnd, ShortToVcc, OpenCircuit) will be realized on the Kl30C HW-Interface (C-Samples).
