  • 7.5.1.39 SYR_DIA_380: check for SW-Update conditions
  • 7.5.1.39.1 Description
  • 7.5.1.39.1.0-1 Brief description: SW-Update (by ESYS or RSU) is not allowed at anytime. BEFORE any SW-Update activities are started, the CCU has to check for "valid conditions" for SW-Update. If "valid conditions" are not existing, the CCU has to deny and SW-Update UDS-requests
  • 7.5.1.39.1.0-2 Preconditions: none (SW-update it triggered by Humans -> this can be requested at any time
  • 7.5.1.39.1.0-3 Trigger: ESYS/RSU-Master request start of SW-Update
  • 7.5.1.39.1.0-4 Input data: Vehicle-Speed
  • 7.5.1.39.1.0-5 Description of behaviour: CCU will not accept SW-Update while car is moving
  • 7.5.1.39.1.0-6 Timing Requirements:none
  • 7.5.1.39.1.0-7 Output data: deny of service
