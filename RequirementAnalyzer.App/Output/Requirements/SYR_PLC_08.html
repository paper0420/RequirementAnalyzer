  • 1.1.30.9 SYR_PLC_08: provide service to enable/disable physical validation while Matching (SLAC) process
  • 1.1.30.9.1 Description
  • 1.1.30.9.1.0-1 Brief description: At present many EVSE in market do not support physical validation, These will me madatory in future. But for meanwhile customer requested this feature to be deactiviated via Diag Job. The function shall be reactivated via Diag Job when needed.
  • 1.1.30.9.1.0-2 Preconditions: NA
  • 1.1.30.9.1.0-3 Trigger: Diag Job
  • 1.1.30.9.1.0-4 Input data: enable or disable feature request
  • 1.1.30.9.1.0-5 Description of behaviour: Default: - Disable the validation during SLAC process - Diag job could read the status - when need Diag jon can reenable this feature.
  • 1.1.30.9.1.0-6 Timing Requirements: standard diagnostic service timing requirement applicable
  • 1.1.30.9.1.0-7 Output data: function is enabled or disabled
  • 1.1.30.9.1.0-8 Postconditions: if disable, no CM_VALIDATE.REQ and CM_VALIDATE.CNF will be sent to charging station (EVSE)
  • 1.1.30.9.1.0-9 Descriptions of exceptions: NA
  • 1.1.30.9.1.0-10 Dependencies and interactions: Charging station can request for Validation in SLAC process, CCU shall respond based on the above set value in this job
