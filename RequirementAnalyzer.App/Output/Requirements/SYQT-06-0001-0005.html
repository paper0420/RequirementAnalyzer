#SYQT-06-0001-0005#

[Charging],[AC_Charging],[AC_Measurement_Accuracy]
Test Objective: HVAC input voltage value confirmation (0 - 300V) (single phase L1, L2)

Requirements

  • 23656 -
    [Not Changed][accepted][SYQT (ENG.10)SYIT (ENG.9)Review][Requirement]
    The mean voltage value (over last 10ms) of the E_U_HVAC_L2Phy shall be sent over signal "HVAC_U_L2Sig" with following accuracy 0V < HVAC_U_L2Sig <300V: +/- 10V Safe State : Respective Qualifier set to not valid, HVAC_U_L2Sig_qi equal to "Invalid" FTT: 160ms
  • 6343 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The sampling rate or bandwidth of the measuring system shall be adapted for the respective test. All measured values shall be recorded with all maximum values (peaks), using the tolerances of the table ENBN_PR_2589
  • 18032 -
    [Not Changed][accepted][Review][Information]
    It shall be guaranteed that occurring voltage peaks of the individual test shall not exceed the measurement range or are not measurable in case of resolution too low.
  • 6474 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The CCU must be designed in a way that it can work propperly in a voltage range from 82 V to 264 V. Remark: These voltages are caused by fluctuations from +10 % to -18 % in the voltage range from 100 V to 240 V.
  • 21980 -
    [Not Changed][accepted][HWT (ENG.7/8)Review][Requirement]
    The charging device must fullfill the following requirements for the complete limited operation range, see section "Operating range of CCU". Differences in the boundary conditions are mentioned explicitely in the requirement text if necessary.
  • 14164 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The one phase charging interface must be devolped for the operation of L1/N AND L1/L2. Information: For the operation between L1 an L2 the maximal applied voltage is 200/240V.
  • 11145 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The specified AC input voltage accuracy shall be complied, taking into account the expected distortion, in accordance with the specified accuracy of the hardware requirements specification (true RMS measurement).
