  • 7.4.2.10 SYR_SUP_090: Supply identification data
  • 7.4.2.10.1 Description
  • 7.4.2.10.1.0-1 Brief description:
  • 7.4.2.10.1.0-2 At any time (e.g. main application OR bootloader OR bootloader-update are running) /any internal state (which will not prevent CAN-Communication) the CCU is able to supply identification data via diagnosis-communication interface via UDS. This data are: - ReadDataByIdentifier ActiveDiagnosticSession - ReadDataByIdentifier ActiveSessionState - ReadDataByIdentifier EnergyMode - ReadDataByIdentifier ExtendedMode - ReadDataByIdentifier FlashTimingParameter - ReadDataByIdentifier ReadGWRoutingStatus - ReadDataByIdentifier MemorySegmentationTable - ReadDataByIdentifier Programming-Counter - ReadDataByIdentifier Programming-Counter-Max-Value - ReadDataByIdentifier ReadMemoryAddress - ReadDataByIdentifier SVK (SVK-Aktuell, SVK-System - Supplier, SVK-Werk) (Steugeräteverbaukennung) This data is used by the diagnosis-tester to check: - If update is possible (newer SWEs are available) - Device is in valid mode to execute an update - Which flashing parameters are to be used - Logical structure of flash-layout - Statistics about flashing
  • 7.4.2.10.1.0-3 Preconditions: none
  • 7.4.2.10.1.0-4 Trigger: request by EDIABAS
  • 7.4.2.10.1.0-5 Input data: UDS service identifier
  • 7.4.2.10.1.0-6 Description of behaviour: requested service/information will be executed/supplied
  • 7.4.2.10.1.0-7 Timing Requirements: none
  • 7.4.2.10.1.0-8 Output data: result of UDS service
