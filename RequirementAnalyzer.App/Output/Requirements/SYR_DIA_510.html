SYR_DIA_510


  • 7.5.1.52 SYR_DIA_510: Save volatile data to NVM -RID (0x4006)
  • 7.5.1.52.1 Description
  • 7.5.1.52.1.0-1 Brief description: The job "RAM_Daten_Sichern" is used to save the data, learned during initilization/vehicle level adjustment, into non-volatile memory without the need for a control unit reset or restart, and therefore to protect the data against process influences such as voltage dips or terminal(klemmen) deactivation.
  • 7.5.1.52.1.0-2 Preconditions: Job shall be accepted in DEFAULT_SESSION APP, EXTENDED_DIAGNOSTIC_SESSION APP, CODING_SESSION APP
  • 7.5.1.52.1.0-3 Trigger: Execution of Diag job 31 01 40 06
  • 7.5.1.52.1.0-4 Input data: data in the ram buffer of 1. BMW_SWC_HvStrNvBlock_NV 2. BMW_SWC_HvChaNvBlock_NV 3. BMW_SWC_HvChaNvBlock_FR
  • 7.5.1.52.1.0-5 Description of behaviour:
  • 7.5.1.52.1.0-5.0-1 The job "RAM_Daten_Sichern" for writing volatile data to the non-volatile memory has the following structure
  • 7.5.1.52.1.0-5.0-2.0-1.0-1 A_Data Byte
  • 7.5.1.52.1.0-5.0-2.0-1.0-2 Parameter Name
  • 7.5.1.52.1.0-5.0-2.0-1.0-3 Hex value
  • 7.5.1.52.1.0-5.0-2.0-2.0-1 '#1
  • 7.5.1.52.1.0-5.0-2.0-2.0-2 RoutineControl Request Service Id
  • 7.5.1.52.1.0-5.0-2.0-2.0-3 31
  • 7.5.1.52.1.0-5.0-2.0-3.0-1 #2 #3 #4
  • 7.5.1.52.1.0-5.0-2.0-3.0-2 sub-function = routineControlType ] routineIdentifier [] = [ byte#1 (MSB) byte#2 ]
  • 7.5.1.52.1.0-5.0-2.0-3.0-3 01 40 06
  • 7.5.1.52.1.0-5.0-3 The blocks that are required to be saved to NVM with this job are BMW_SWC_HvStrNvBlock_NV BMW_SWC_HvChaNvBlock_NV BMW_SWC_HvChaNvBlock_FR as per the alignment with BMW. The whole content of these blocks shall be stored NVM on request of this job.
  • 7.5.1.52.1.0-5.0-4 If the process has been started successfully, the control unit shall immediately output the following positive response 71 01 40 06
  • 7.5.1.52.1.0-5.0-5 The saving data may take a long time, a status query shall be used to find out whether or not the save process is already complete. This is performed with the following request â€•RAM_Daten_Sichern_Status 31 03 40 06
  • 7.5.1.52.1.0-5.0-6 The control unit shall respond to a successful status query with the following positive response 71 03 40 06 XX XX has the following values 0x00 - Writing successful 0x01 - Writing failed 0x02 - Writing ongoing 0x03 - Writing not yet started (Routine not started) 0x04 - 0xFF - Reserved
  • 7.5.1.52.1.0-6 Timing requirements: The maximum execution time of the service "RAM_Daten_Sichern" shall not exceed 4 seconds. The data amount shall be reduced such that the maximum execution time is not exceeded.
