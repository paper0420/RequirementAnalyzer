  • 8.1.55.1.0-5 Description of behaviour: Event signals are supported only trhough Queued interface. For Tx Events the Queue has to be written only when an event occurs - So application has to ensure the proper handling of Queue so that Queue is not overflown when more event happens less than deboucne time. For Rx Signal ComAdapter will write into Queue everytime the Event signal is received , ComAdapter polling at Event Time ensures the Queue is not overflowing - but application must ensure the queue is read every Debounce time atleast or faster to avoid a queue overflow.
