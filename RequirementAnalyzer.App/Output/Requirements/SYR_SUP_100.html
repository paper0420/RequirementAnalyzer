  • 7.4.2.11.1 Description
  • 7.4.2.11.1.0-1 Brief description:
  • 7.4.2.11.1.0-2 This Use-case will use cryptographic methods to ensure: a) Diagnosis-tester is valid device to send new data to flash-memory b) The CCU is valid device to receive data from diagnosis-tester c) The data transferred from diagnosis-tester to CCU is valid/authentic/unmodified SHA-256 The case a) will be ensured by “Security Access” in UDS-style. The CCU will send “seed” on request and diagnosis-tester will calculate its answer based on this “seed”. If the “answer “correct, the CCU will enable sw-update. See Chapter 4.6 of [AUS] The case b) [sic!] will be ensured by symmetric-encryption of data to be written. Panasonic will encrypt all SWEs internally with one, common AES-256 Key for all devices. The key for decryption is located in secure memory of CCU. Without this key, decryption is impossible. This is decision taken by Panasonic to protect Panasonic-HW against misuse. Related UDS Services: - SecurityAccess RequestSeed - SecurityAccess SendKey - DiagnosticSessionControl
