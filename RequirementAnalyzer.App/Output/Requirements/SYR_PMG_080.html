SYR_PMG_080


  • 7.2.1.7 SYR_PMG_080: enable CAN wakeup
  • 7.2.1.7.1 Description
  • 7.2.1.7.1.0-1 Brief description:
  • 7.2.1.7.1.0-2 This UC will detect if CAN-Wakeup has to be enabled (=CAN-communication / CAN wake up line) and enable it. This includes the correct configuration of hardware (CAN-Transceiver / CAN-wake up line) and software. This UC is intended to prepare the wake up from “deep sleep” of CCU, not wake up while system-cpu is running. If wake up happens while System-CPU is running, it will be processed by software, only. The wakeup will be enabled, if CCU prepares for “sleep-mode”. The CAN-Transceiver will be set to “sleep-mode”
  • 7.2.1.7.1.0-3 Preconditions: CCU prepares to enter CAN-Sleep
  • 7.2.1.7.1.0-4 Trigger: shutdown of CAN-Networkmanagement
  • 7.2.1.7.1.0-5 Input data: none
  • 7.2.1.7.1.0-6 Description of behaviour: the CAN-HW (Controller/Transceiver/SSWU) will be configured to detect and handle CAN-Wakeup (ATTENTION: Depending on dynamic shut-down sequence, the CAN-Wakeup can/must be detected with different methods: CAN-Controller -> SSWU -> Transceiver) Attention: some CAN cannot be waken up be Transceiver, this will be done by discrete wake-up line
  • 7.2.1.7.1.0-7 Timing Requirements: none, since shutdown-proceduere has no timing constrains
