SYR_CIN_270


  • 8.1.31 SYR_CIN_270: Provide E/E support for Kafas by coding parameter
  • 8.1.31.1 Description
  • 8.1.31.1.0-1 Brief description: Kafas control unit is not present in All Vehicle variants. Based on CODING Parameter for Kafas the PDUs which are to be received from "KAFAS" has to be Disabled or Enabled in CCU.
  • 8.1.31.1.0-2 Preconditions: NA
  • 8.1.31.1.0-3 Trigger: NA
  • 8.1.31.1.0-4 Input data: CAF , BMW NK
  • 8.1.31.1.0-5 Description of behaviour: Based in Coding parameter KafasAvail having possible value "verbaut " or "nicht verbaut" the Kafas will be present in the Vehicle. So Based on CODING Parameter monitoring of Kafas related PDUs in CCU has to be Enabled or Disabled. Acceptance : - Coding accesible to SWC - No Timeout, CRC , Alive Error monitoring for PDUs from Kafas when Kafas is not Available - Timeout , CRC, Alive Monitoring and Signal Values are provided to SWC for PDUs from Kafas when Kafas is Available.
  • 8.1.31.1.0-6 Timing Requirements: NA
  • 8.1.31.1.0-7 Output data: NA
  • 8.1.31.1.0-8 Postconditions: Acceptance : - Coding accesible to SWC - No Timeout, CRC , Alive Error monitoring for PDUs from Kafas when Kafas is not Available - Timeout , CRC, Alive Monitoring and Signal Values are provided to SWC for PDUs from Kafas when Kafas is Available.
  • 8.1.31.1.0-9 Descriptions of exceptions: NA
  • 8.1.31.1.0-10 Dependencies and interactions: The List of PDUs from CSM cannot be found out by CCU as CSM is routing the PDUs through BDC/ZGW. BMW is responsible for notifying Panasonic if a new Receive PDU for CSM is to be added for CCU.
