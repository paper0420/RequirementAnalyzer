  • 7.5.1.48 SYR_DIA_470: [G08] Disabling DTCs 0x023A80, 0x023A90, and 0x023A91 in BEV18 Line
  • 7.5.1.48.1 Description
  • 7.5.1.48.1.0-1 Brief description: The DTCs 0x023A80, 0x023A90, and 0x023A91 can be disabled in BEV18 line.
  • 7.5.1.48.1.0-2 Preconditions:
  • 7.5.1.48.1.0-3 Trigger:
  • 7.5.1.48.1.0-4 Input data: A calibration parameter "b_SecOCActive" must be set by using BMW CRETA tool for the required vehicle variant.
  • 7.5.1.48.1.0-5 Description of behaviour: The following DTCs can be disabled by using the alibration value "b_SecOCActive" in BEV18Line: 0x023A80 ZERTIFIKATE_UND_BINDINGS_TYP1_WERK_NICHT_BEREIT, 0x023A90 SecOC: Keypackprüfung fehlgeschlagen, 0x023A91 SecOC: Bypass aktiv - Sichere Onboard Kommunikation deaktivert For the DTCs 0x023A80, 0x023A90, and 0x023A91, ActiveConditon "SysStateSecOcActive" was added into ZEDIS V20.1. Implementation Details: b_SecOCActive is a calibration parameter available in a2l and will be modified in BMW CRETA tooling depending on vehicle variant Logic :  value 0 : SecOC is NOT activated in the CCU => the SecOC DTCs shall be deactivated and thus the enabling condition SysStateSecOcActive shall be set to FALSE  value 1 : SecOC is activated in the CCU => the SecOC DTCs shall be deactivated and thus the enabling condition SysStateSecOcActive shall be set to TRUE Polling of b_SecOCActive is not necessary as value will not be changed dynamically. (only changes in CRETA expected). Please check b_SecOCActive only at start-up to improve runtime.
