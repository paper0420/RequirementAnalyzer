  • 6.1.13.2 Differences to CRS
  • 6.1.13.2.0-1 The cycle time 10ms is not specified in the KLH for the LvDCDC measurement of the HV-input current. Explaination, why it shall be10ms: The 10ms rate is mentioned in KLH 23931 for the signal E_I_DCDC. E_I_DCDC can be assumed to more the output current of the LvDCDC. For LT4 HV current the 10ms is mentioned in KLH 23665. Then the 10ms is required in KLH 23644 for E_U_LT4. All the requirements can lead to the assumption that the normal cycle time is 10ms. Panasonic development center in Japan has written in the Jira Ticket CCU-209 anyway that there is a monitoring cycle of 10ms: [Refinement] Input : hwi_adc_HV_current Output : Measured_HVcurrent Monitoring cycle : 10[ms] Pre condition : none (always)
