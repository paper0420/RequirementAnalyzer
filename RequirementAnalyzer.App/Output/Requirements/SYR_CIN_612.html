  • 8.1.67 SYR_CIN_612 : New partional network request for climatization
  • 8.1.67.1 Description
  • 8.1.67.1.0-1 Brief description: For the climatization of the high voltage battery a new partial network request will be introduced to reduce the activity time of other ECU's which are not needed in this case. The new partial should be requested via NM3Request by the BSW, if BMW_st_ComNetReq Bit4 is set to1 (=16). Tx side : Comnetreq to be connected to the NM3 Tx PDU : Rx side : This new Partial Network will be received from BCP/ZGW NM3 (0x510). This new PNC has to be configured acc. to the boardnet.arxml Lin Communication shall be active when this PNC is requested from BCP/ZGW. This can be done by changing the application parameter in CRETA(PnSelectCalibration_C).
  • 8.1.67.1.0-2 Preconditions: CCU is operational.
  • 8.1.67.1.0-3 Trigger: Charging is running in partial network. HV battery gets warm and requests cooling.
  • 8.1.67.1.0-4 Input data: NK integration for New partial network request for climatization
  • 8.1.67.1.0-5 Description of behaviour: Tx: The new partial network shall be requested via NM3Request(0x53A) by the CCU, if BMW_st_ComNetReq Bit4 is set to 1 (=16). Rx: This new Partial Network will be received from BCP/ZGW NM3 (0x510). CCU shall enable respective Tx and Rx Messages. LIN Communication shall also active based upon the PNC active state.
  • 8.1.67.1.0-6 Timing Requirements: NA
  • 8.1.67.1.0-7 Output data: CAN messages defined in ECU-extract shall be transmitted or recieved based upon th PNC activation.
  • 8.1.67.1.0-8 Postconditions: None
