SYR_DIA_400


  • 7.5.1.41 SYR_DIA_400: detect and judge LIN errors
  • 7.5.1.41.1 Description
  • 7.5.1.41.1.0-1 Brief description: By restrictions of LIN-Hardware the CCU has to derate the LIN-Error-Detection mechanism. Since the CCU is the "LIN-Master" it has to be able to detect 3 different kinds of LIN-Errors: 1) broken cable: It's impossibe to send/receive messages to/from slaves, because the cable is broken 2) broken slave: The message reaches the slave physically, but the slave is not able to send answer (=slave broken) 3) sporadic communication failes: If any slave detects communication problem (e.g. CRC/Framing) in received frame from Master, the slave will inform this within his next frame, send to master by "Error-Bit")
  • 7.5.1.41.1.0-2 Preconditions: LIN communication is active/should be active
  • 7.5.1.41.1.0-3 Trigger: any LIN frame (received or missing)
  • 7.5.1.41.1.0-4 Input data: data from MCAL-Lin-Driver / Lin-Protocol-Stack
  • 7.5.1.41.1.0-5 Description of behaviour: 1) cable: If LIN-Driver indicates "no physical transmission possible" -> Set DTC "LIN communication fault" and SUPRESS case 2 and 3 DTCs 2) broken slave: if no frame from slave has been receibed -> Set DTC "Timeout Slave XXX" 3) slave detected problem: If frame from slave with set "response error bit" -> Set DTC "Lin communication error" if error is not detected/informed anymore, the related DTC shall be unset
  • 7.5.1.41.1.0-6 Timing Requirements: depends on LIN / see ZEDIS-export
  • 7.5.1.41.1.0-7 Output data: related DTC
  • 7.5.1.41.1.0-8 Postconditions: none (this UC will never terminate)
  • 7.5.1.41.1.0-9 Descriptions of exceptions: none
  • 7.5.1.41.1.0-10 Dependencies and interactions: none
