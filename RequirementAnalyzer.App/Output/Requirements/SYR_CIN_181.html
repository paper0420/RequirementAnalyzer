SYR_CIN_181


  • 8.1.22 SYR_CIN_181 : Control LIN Communication
  • 8.1.22.1 Description
  • 8.1.22.1.0-1 Brief description: LIN Communication has to be stopped when Flashing is happening LIN Communication has to be stopped when None of the Lin COmponent Driver Application SW-C needs communiation. LIN Communication has to be stopped before CCU Shutdown (CAN-FD == NO_COM). LIN communication has to be present when CCU is running(CAN-FD == FULL_COM) and Flashing is not happening and LIN COmponent Driver Application SW-C needs communiation.
  • 8.1.22.1.0-2 Preconditions: none
  • 8.1.22.1.0-3 Trigger: Wakeup , Interfaces from LIN Application Component Driver SW-Cs , Fashing , Sleep
  • 8.1.22.1.0-4 Input data: Lin Component Driver Application SW-C Interfaces indicating need for communication. CAN-FD Communication status can be used to determine Sleep Readiness of CCU. Energy Mode for Flashing will be available from BAC module. LIN Communication stays in FULL Communication for a Minumum Time of 1.2 second before moving to LIN Sleep.
  • 8.1.22.1.0-5 Description of behaviour: Refer svn documentation for detailed explanation: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Documentation/COMAdapter/00_CRs_Tasks/CR827_CCU-23352_V3_31072020.pptx
  • 8.1.22.1.0-6 Timing Requirements:
  • 8.1.22.1.0-7 Output data: LIN Communication State. Debug message showing reason for LIN Active Communication.
  • 8.1.22.1.0-8 Postconditions: When LIN Communication is Active - LIN Transmit and Reception on LIN is possible as in SYR_CIN_180 - Error monitoring (Timeout) for LIN PDUs are possible. - LIN Communication fault detection possible When LIN Communication is not Active - LIN trasnmit and Recetion of LIN PDUs not possible. - Error Monitoring (Timeout) for LIn PDUs not possible - LIN Communication Fault detection not possible.
  • 8.1.22.1.0-9 Descriptions of exceptions: none
  • 8.1.22.1.0-10 Dependencies and interactions: none
