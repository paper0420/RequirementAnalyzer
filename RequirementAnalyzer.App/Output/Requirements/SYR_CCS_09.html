SYR_CCS_09


  • 1.1.31.13 SYR_CCS_09: provide SAScheduleList to bmw ecu for charging profile
  • 1.1.31.13.1 Description
  • 1.1.31.13.1.0-1 Brief description: SAScheduleTuple received in ChargeParameterDiscoverRes containing PMaxschedule and SalesTariff data shall be provided to bmw ecu (e.g. MGU) for calculation/selection of chargeprofile. which will be send back to CCU. This selected SAScheduleTuple will provided to Charging station in PowerDeliveryReq and will be used as current excecussion profile. During the sleep cycle of CCU, care must be taken that selected SAScheduleTuple shall be saved for future use (e.g. charge pause senario)
  • 1.1.31.13.1.0-2 Preconditions: receiving chargeparameterdiscoveryres from charging station
  • 1.1.31.13.1.0-3 Trigger: bmw trigger to send SAScheduleTuple via BMW_st_CtlTranSAScdList
  • 1.1.31.13.1.0-4 Input data: data to be sent from CCU
  • 1.1.31.13.1.0-5 Description of behaviour: The latest sequence for communication between CCU and MGU over Some/IP and interaction at CCU with ISO sequence is described below: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/Sequence/LH_SOME_IP_CCU_13462_CCU_16475.pdf
  • 1.1.31.13.1.0-6 Timing Requirements: refer the sequence as linked
  • 1.1.31.13.1.0-7 Output data: after transmission in form bmw swc via BMW_st_ActStTranSAScdList signal
  • 1.1.31.13.1.0-8 Postconditions: Communincation between CCU and ECU (MGU) MUST be established
  • 1.1.31.13.1.0-9 Descriptions of exceptions: all error scenario is not discussed. if CCU fails to transmit data to BMW ECU inform bmw swc and wait for next instruction. Possible behaviour from CCU: In Case of 5%: we will fail on first failed transmission, BSW shall send SessionStop and terminate TCP(HLC). In Case of nominal dutycycle: BMW SWC can fallback to BS in case MGU failed to provide CharingProfile, which means no input for BSW for PowerDeliveryReq. BSW shall send SessionStop and terminate TCP
