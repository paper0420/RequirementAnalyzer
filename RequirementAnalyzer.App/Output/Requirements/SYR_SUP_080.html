  • 7.4.2.9 SYR_SUP_080: Update data/software via UDS
  • 7.4.2.9.1 Description
  • 7.4.2.9.1.0-1 Brief description:
  • 7.4.2.9.1.0-2 This Use-Case is the master Use-Case for diagnosis (UDS) driven update of data and software. The update is driven and supplied by external diagnosis tester. The whole communication is based on UDS-Services. This is the single method for update in service and production of CCU. This update is protected against unauthorized update of any data by and device. Only valid diagnosis-tester are able to write valid data into CCU. This is ensured by methods, which are described in “Security specification” [TBD].
  • 7.4.2.9.1.0-3 Preconditions: none
  • 7.4.2.9.1.0-4 Trigger:external applied SW-update request
  • 7.4.2.9.1.0-5 Input data: software as binary to flash
  • 7.4.2.9.1.0-6 Description of behaviour: new software will be flashed into CCU's internal flash memory
  • 7.4.2.9.1.0-7 Timing Requirements: finished within < 5min
  • 7.4.2.9.1.0-8 Output data: none
  • 7.4.2.9.1.0-9 Postconditions: software of CCU was updated
