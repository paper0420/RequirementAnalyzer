  • 7.2.1.3.1.0-2 The CAN wakeup can be requested by “CAN activity” or “CAN wakeup line”. Depending on current system-state the processing of wake-up events is different. This UC does NOT describe the application-communication itself, it just describe the wake up / shut down of CCU. If the CCU has no communication demand/request, it will enter “sleep-mode”. This UC will handle the CAN-wake up event (discrete line and/or CAN-wakeup) and bring the CCU into a state which enables the software to operate these communication lines. 1.1. SYR_PMG040 detect and handle wakeup via EVSE This UC will handle the wakeup via discrete lines from EVSE.
