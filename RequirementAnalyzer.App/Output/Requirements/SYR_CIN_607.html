SYR_CIN_607


  • 8.1.62 SYR_CIN_607: Wildcard for LIN Master Request
  • 8.1.62.1 Description
  • 8.1.62.1.0-1 Brief description: If the supplier ID and/or the function ID is not known, the wildcard(s) shall be used.
  • 8.1.62.1.0-2 Preconditions: CCU is running and LIN wake-up happens.
  • 8.1.62.1.0-3 Trigger: After LIN wake-up, master request for each slave shall be sent on bus in between LIN run_main execution.
  • 8.1.62.1.0-4 Input data: Data for creating LIN master request for each slave.
  • 8.1.62.1.0-5 Description of behaviour: Some LIN Slaves are not responding to the master Request as the supplier Id and function Id are defined specifically to one waterpump supplier. With this change, the target is to introduce the Wildcard as per Lin specification for: set supplied ID : 0x7FFF set function ID : 0xFFFF Change shall be done for all request type defined for only water-pump slaves EWP22 & EWP23 (i.e. Seriennummer, Produktionsdatum, BMW-Teilenummer, Software-Referenznummer, Hardware-Referenznummer, LIN-Identification data, Firmware-Referenznummer (not in scope) & Variantenindex (not in scope)) Refer sheet for data required for LIN master request: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Documentation/Thermal_Mgmt/LIN_Meta_Data_Requests.xlsx [Out of scope] Firmware-Referenznummer (no such request is sent by CCU): EWP22: 5A 06 B2 2C FF 7F FF FF EWP23: 4B 06 B2 2C FF 7F FF FF Variantenindex (no such request is sent by CCU): EWP22: 5A 06 B2 2D FF 7F FF FF EWP23: 4B 06 B2 2C FF 7F FF FF
  • 8.1.62.1.0-6 Timing Requirements: none
  • 8.1.62.1.0-7 Output data: LIN slave shall respond correctly to Master Request.
  • 8.1.62.1.0-8 Postconditions: LIN sensor ident. lesen job shall provide data for waterpump slaves and LIN RTE interfaces (if any) shall be updated.
  • 8.1.62.1.0-9 Descriptions of exceptions: none
  • 8.1.62.1.0-10 Dependencies and interactions: Autosar stack which are involved in the LIN\CAN Communication
