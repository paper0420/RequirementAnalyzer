#SYQT-24-0001-0001#

[Charging],[Component_Protection],[AC_Undervoltage]
Test Objective: Stopping AC charging at AC low voltage detection (Single phase L1)

Requirements

  • 18047 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    An undervoltage of the local low voltage supply is to be signaled with an entry in the error memory.
  • 18393 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The "Komponentenschutz und Derating" function shall detect the following protection conditions (detect errors and monitor limit values) in order to adjust operation of the charging electronics without damaging the hardware: - AC undervoltage - DC-HV undervoltage - AC overvoltage - DC-HV overvoltage - Maximum AC current - Maximum DC-HV current - Hardware fault (e.g. short circuit) - Excess temperature - Broken N-Line
  • 18401 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The function "Komponentenschutz und Derating" shall detect the AC undervoltage error if the charger measures an AC voltage below the AC voltage operating limit (e.g. 85 V AC) for a parameterizable period (e.g. 120 sec).
  • 18402 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    If an AC undervoltage is detected, the function "Komponentenschutz und Derating" shall set the "Abschaltung Unterspannung AC" bit in the error status word.
  • 18403 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The diagnostics for AC undervoltage shall be performed only when the target operating mode "Charge" (2) is commanded by the "LDK" function.
  • 11305 -
    [Not Changed][partly accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    The "Ansteuerung_Ladegerät_AC/DC_Hochvolt" function shall assign the following information to the "Ist_Grund_Limit_Ladegerät-AC/DC" signal: 0--- ---1 ---- ---- AC voltage is missing 0--- --1- ---- ---- No power delivered, but operation mode "Charging" requested. 0--- -1-- ---- ---- Other reasons
