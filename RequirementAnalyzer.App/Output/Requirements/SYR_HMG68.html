  • 3.1.12 SYR_HMG68: AKKS - INIT
  • 3.1.12.1 Description
  • 3.1.12.1.0-1 Brief description: Every time the AKKS sends an INIT Request (ST_INIT_AKKS_LIN=1) the CCU has to perform a proper INIT Procedure
  • 3.1.12.1.0-2 Preconditions: LIN Wake-Up and ST_INIT_AKKS_LIN=1
  • 3.1.12.1.0-3 Trigger:ST_INIT_AKKS_LIN=1
  • 3.1.12.1.0-4 Input data: The only relevant input data to the CCU concerning this functionality is ST_INIT_AKKS_LIN
  • 3.1.12.1.0-5 Description of behaviour: 1.Soon after the boot: a. the AKKS does not know its current position which is not permanently stored in its EEPROM ----> the AKKS typically sends a default value of its current position which is nevertheless not coherent with the position reference system which has to be used by the CCU (for ex. the current position of the flap can be 0° but the AKKS send the corresponding LIN Value for a different angle value ); b. the CCU will send the default target position of 10000, which corresponds to the 0° position (SMOT_PO_AKKS_LIN=1); 2. the AKKS will send an INIT Request (ST_INIT_AKKS_LIN=1) to the CCU; 3. soon after the reception of the INIT Request, the CCU will send a Carry-over (SMOT_PO_COOV_AKKS_LIN=1) for a time length of at least 2 LIN-Cycles; 4. the AKKS will take over the target position as its current position, but without moving its flap ----> (ST_SMOT_AVL_PO_AKKS_LIN=SMOT_PO_AKKS_LIN==10000)
  • 3.1.12.1.0-6 Timing Requirements: provided that BMW has specified no timing constraints concerning the above steps mentione in the section "Description of behaviour", these are the typical timing values of the INIT procedure: - The AKKS typically sends the INIT request at the first LIN-Cycle after the Wake-Up, i.e. after a time t < 400ms after the LIN Wake-Up (step 2.); - Max. 2 LIN Cycles (800ms) after the sending of the INIT Request by the AKKS, the CCU will send the Carry-Over (step 3.) - The AKKS will typically take-over the target position as its current position within 2 LIN-Cycles (800ms)
  • 3.1.12.1.0-7 Output data: The relevant output data sent by the CCU concerning this functionality are: - SMOT_PO_AKKS_LIN (target-position) - SMOT_PO_COOV_AKKS_LIN (Carry-Over)
  • 3.1.12.1.0-8 Postconditions:current position ==target position == 10000 ----> ST_SMOT_AVL_PO_AKKS_LIN==SMOT_PO_AKKS_LIN==10000
  • 3.1.12.1.0-9 Descriptions of exceptions: in case the AKKS will not take over the target position sent by the CCU., the AKKS will not work properly ---> this will handled by keeping the Carry-Over set to 1 until ST_SMOT_AVL_PO_AKKS_LIN==SMOT_PO_AKKS_LIN==10000 (not implemented in I300, starting from I310 it will be introduced a DTC in case of INIT failure); AKKS1 is not mounted for SP21 car lines.
  • 3.1.12.1.0-10 Dependencies and interactions: interaction and dependency is with: - the whole LIN-network ----> in case of physical failure on the LIN no INIT procedure can be performed; - the AKKS itself -----> since the AKKS has to be able to take over the target position during the INIT .
