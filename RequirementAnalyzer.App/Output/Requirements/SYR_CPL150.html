  • 1.1.27 SYR_CPL150: security shutdown of SCC in case of attack
  • 1.1.27.1 Description
  • 1.1.27.1.0-1 Brief description: The SCC protocoll has strict handshake, driven by CCU. If any deviation in this handshake happens, the CCU will cancel any SCC communication. This is security-requirement! Communication via SCC will be started again after complete "disconnect/connect" cycle. ( See EA-Modell!)
  • 1.1.27.1.0-2 Preconditions: SCC connection established
  • 1.1.27.1.0-3 Trigger: deviation of SCC-handshake/protocoll
  • 1.1.27.1.0-4 Input data: SCC-datagramm
  • 1.1.27.1.0-5 Description of behaviour: SCC will be stopped in case of handshake/protoll deviation
  • 1.1.27.1.0-6 Timing Requirements: none
  • 1.1.27.1.0-7 Output data: none
  • 1.1.27.1.0-8 Postconditions: SCC communication is disabled
  • 1.1.27.1.0-9 Descriptions of exceptions: none
  • 1.1.27.1.0-10 Dependencies and interactions: none
