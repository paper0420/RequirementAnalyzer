  • 8.1.32 SYR_CIN_280: LIN Communication Fault
  • 8.1.32.1 Description
  • 8.1.32.1.0-1 Brief description: The CCU is LIN-Master, by this the CCU must supervise the LIN-Communication with LIN-Slaves (fan/water pump/actuators).
  • 8.1.32.1.0-2 Preconditions: LIN is operational
  • 8.1.32.1.0-3 Trigger: Physical Error on LIN Bus (- Header not sent due to Short-Circuit to GND / Short-Circuit to BAT / Open Circuit)

  • 8.1.32.1.0-5 Description of behaviour: CCU should be able to detect physical LIN bus error (ex. Header cannot be sent due to Shortcut). Motivation is to raise LIN Communication Fault DTC and to suppress multiple timeout DTCs. CCU is required to detect the FSE11 shown in requirement LIN_SYS_1720 in LH 10306939-000-03 CCU – LIN HW platform is not supporting detection of LIN_E_TIMEOUT (Timeout caused by Hardware Error) Fault. Vector Lin_Core provides LIN_E_TIMEOUT only if HW platform supports it. BMW Expectation – If communication on LIN Bus has an Short-circuit to Gnd / Short Circuit to Battery (FSE11), CCU should detect an Error and this Error Memory Entry must be used to suppress further reporting of Timeout Error for each LIN Slave nodes. CCU – Software workaround solution to be done to detect the FSE11 (LIN_SYS_1720) LIN Controller (Master Mode) support flagging of Bit Error (value read back from Bus is different from the value transmitted.) Restriction agreed with BMW: Since there is NO HW Error Detection possible, we want to utilize OSI-Layer2 error-detection as indicator for Layer1 problems/limitations. HW Fault distinguishing is not possible. This would only mean there is problem with LIN (but not with CCU). The fault detection is possible only when LIN is Operational.
  • 8.1.32.1.0-6 Timing Requirements: Calibration Timer is 800 ms (< 5 cycle time for timeout DTCs)
