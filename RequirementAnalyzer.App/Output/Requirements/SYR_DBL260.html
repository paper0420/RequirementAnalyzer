SYR_DBL260


  • 7.6.1.26 SYR_DBL260: pointer based calibration - update of working page mapping
  • 7.6.1.26.1 Description
  • 7.6.1.26.1.0-1 Brief description: The mapping of working page records is done via selectionMap. Update of selectionMap is a standard calibration transaction. Since the selectionMap will always be available in working page it can be directly influenced by direct access without any indirection via pointer table. or definition and terms see also: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.03_SystemArchitecturalDesign/ENG.03_SystemArchitecturalDesignSpecification/AdditionalDocuments/Calibration/bmw_ccu_calibration_architecture.pptx
  • 7.6.1.26.1.0-2 Preconditions: XCP session established.
  • 7.6.1.26.1.0-3 Trigger: XCP calibration write command to selectionMap with new mapping content.
  • 7.6.1.26.1.0-4 Input data: new mapping content for selectionMap
  • 7.6.1.26.1.0-5 Description of behaviour: (1) XCP slave checks consumption of working page and updates onlineCalibDataFillLevel and onlineCalibDataSize. If working page is full (onlineCalibDataFillLevel exceeds 100%) the operation is aborted with error failure. No changes shall be performed. Only if onlineCalibDataFillLevel below 100%: (2) XCP slave writes new content into working page selectionMap (3) XCP slave updates onlinePointerTable based on new selectionMap content (4) XCP slave copies data from reference page to working page data area using modified onlinePointerTable.
  • 7.6.1.26.1.0-6 Timing Requirements: command execution shall occur at low priority
  • 7.6.1.26.1.0-7 Output data: success information
  • 7.6.1.26.1.0-8 Postconditions: selectionMap in working page updated; onlinePointerTable updated; calibration data copied from reference page to working page
  • 7.6.1.26.1.0-9 Descriptions of exceptions: If working page is overloaded the operation is refused.
  • 7.6.1.26.1.0-10 Dependencies and interactions: None.
