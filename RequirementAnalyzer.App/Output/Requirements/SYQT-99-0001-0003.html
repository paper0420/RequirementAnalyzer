#SYQT-99-0001-0003#

[Functional_Safety],[Functional_Safety],[Functional_Safety]
Test Objective: HVDC output current is limited to limit value (1 phase) *) Details Calibration value of over current threshold is 45 A FTT 1s inform BMW SW

Requirements

  • 23666 -
    [Not Changed][partly accepted][SYQT (ENG.10)SYIT (ENG.9)][Requirement]
    While AC charging, the CCU shall limit the output current on HVDC (H_I_LT4) as follows: - 17A for 3,5KW charger - 33A for 7KW charger - 52A for 11KW charger - 52A for 15KW charger - 70A for 22KW charger (Calibratable) FTT: 1000ms Safe state: Disable the AC charging and inform BMW SW Note: It is assumed that no loads are present on the CCU (LT5,LT7 and DC/DC current consumption equal to 0A), then above values refer to AC/DC converter Note: The current limit is depends on cables rated current value (Relaxed for 22KW)
  • 18395 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The "Komponentenschutz und Derating" function shall stop operation of the charging electronics when a protection condition is present.
  • 18397 -
    [Not Changed][accepted][SYQT (ENG.10)][Requirement]
    The "Komponentenschutz und Derating" function shall trigger an operating mode change to the "Error" state when a protection condition is present.
  • 18399 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    If a self-protection emergency deactivation takes place, an entry shall be made in the error memory clearly identifying the cause of the error.
  • 18422 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    If a DC overcurrent is detected, the function "Komponentenschutz und Derating" shall set the "Abschaltung Maximalstrom DC-HV" bit in the error status word.
  • 18423 -
    [Not Changed][accepted][SYQT (ENG.10)SWQT (ENG.8)][Requirement]
    The diagnostics for DC-HV overcurrent diagnostics shall be performed in every operating state.
