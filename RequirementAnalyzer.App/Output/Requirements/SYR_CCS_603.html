  • 1.1.31.17 SYR_CCS_603 : Provide Intermediate Fallback Solution for ISO 15118 AC EIM Sequence
  • 1.1.31.17.1 Description
  • 1.1.31.17.1.0-1 Brief description: CCU shall be able to communicate with charging station following ISO AC-EIM 15118 sequence even if communication between CCU & MGU is not possible. This means CCU is not capable of comparing different SASchedule but only proceed with default SASchedule.
  • 1.1.31.17.1.0-2 Preconditions:Flap opened. Vehcile is connected to Charger. Proximity detection.
  • 1.1.31.17.1.0-3 Trigger:EVSE applies valid duty-cycle(5%) on Control Pilot-Line and LIM-C initiates the communication
  • 1.1.31.17.1.0-4 Input data:analog values of Pilot-Line (duty-cycle/voltage level), proximity-line voltage level, PLC-communication of Pilot-Line.
  • 1.1.31.17.1.0-5 Description of behaviour:[Current Design SYR_CCS_15 & SYR_CCS_09]For AC EIM charging sequence, EVSE sends ChargeParameterDiscovery response which then send by CCU to MGU. Charging Profile and SAScheduleTupleID is calculated and provided by MGU to CCU. BMW SWC provides this data over RTE interface to PANA SWC and using the data, AC EIM charging sequence is processed from Power Delivery Request. In case of failure between CCU & MGU (example, performance timer expired or SOME/IP socket issues or memory issue), CCU shall switch to basic charging (controlled by BMW SWC). [Change in scope of CR 921 / current SYR] Supplier shall implement intermediate fallback solution for AC EIM. https://asc.bmw.com/jira/secure/attachment/2533909/CCU-27169_AC-EIM_Fallback_Solution_v1.pdf Overview on fall back strategy: https://asc.bmw.com/jira/secure/attachment/2533909/Fallback_Scenario.png Referring to ISO15118-2 requirements, [V2G2-297] The first SAScheduleTuple element in the SAScheduleListType shall be defined as default SASchedule. [V2G2-298] If the EVCC is not capable of comparing different SAScheduleTuple elements or comparison fails, the EVCC shall choose the default SAScheduleTuple according to [V2G2-297]. During charging session, if communication between CCU & MGU is not possible/or failed (such as Socket connection issue detected at CCU, Subscription not received by CCU, SOME/IP event is not acknowledged at CCU or Response not received from MGU to BMW SWC) is detected and due to which optimized Charging Profile and selected SAScheduleTupleId cannot be provided to Charging station in Power Delivery Request. In this case, CCU shall follow V2G-297 and V2G2-298 and shall continue charging session for AC EIM. Based on Qualifier value set by BMW SWC, PANA BSW should extract the first Schedule ID from ChargeParameterDiscovery Resp and send this back to the EVSE as selected SAscheduleID in Power Delivery Request. Charging Profile is optional parameter in Power Delivery Request so it will not be send.

  • 1.1.31.17.1.0-8 Postconditions:AC EIM based charging session is established between EV and EVSE (w/o charging profile and with default SAScheduleId).
