  • 7.4.2.17 SYR_SUP_160: write data to flash
  • 7.4.2.17.1 Description
  • 7.4.2.17.1.0-1 Brief description:
  • 7.4.2.17.1.0-2 This Use-Case will write data from Tester/INCA to physical flash memory. If virtual addressing is used, this Use-Case will map the virtual addresses to physical addresses. This Use-Case will process hardware related issues, generated by the flash-memory (re-tries). Since the CCU has no file-system, hw-problems of flash-memory cannot be solved by this Use-Case; at least this Use-Case will detect and notify any problem which prevents correct storage of data into flash memory. In addition this Use-Case is responsible to transfer the data to be flashed in “offshore” flash memory (e.g. Flash of DSP / PIC). This Use-Case will ensure same operation like on “system cpu” for the “offshore”-CPUs.
  • 7.4.2.17.1.0-3 Preconditions: none
  • 7.4.2.17.1.0-4 Trigger: request to persist data
  • 7.4.2.17.1.0-5 Input data:data to be stored
  • 7.4.2.17.1.0-6 Description of behaviour: the referred data will be written into flash memory
  • 7.4.2.17.1.0-7 Timing Requirements: none (controlled by HW = static attribute)
  • 7.4.2.17.1.0-8 Output data: result of flash-operation (PASSED/FAILED)
  • 7.4.2.17.1.0-9 Postconditions: flash memory has been written
