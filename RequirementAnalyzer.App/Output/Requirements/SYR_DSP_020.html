  • 7.4.1.9 SYR_DSP_020: flash update application
  • 7.4.1.9.1 Description
  • 7.4.1.9.1.0-1 Brief description:
  • 7.4.1.9.1.0-1.0-1 The “flash update” of DSP is handled by “Flash Bootloader”. The “Flash bootloader” will be started on request of the “bootloader” in “system CPU”. The bootloader in “system cpu” will execute complete UDS-communication with diagnosis-tester and extract DSP-related data. By this the “flash bootloader” of DSP has very simple, atomic interface: - Delete application program - Write new data to application-program memory (physical addressing) - Verify written data - Restart DSP (-> new application program) This is STATELESS interface, all states are handled by “bootloader” in “system cpu”. In case of failure (verification failed), the “bootloader” in “system cpu” is in charge to retry.
  • 7.4.1.9.1.0-2 Preconditions:
  • 7.4.1.9.1.0-2.0-1 The device is in one of the following system states: DSP was requested to enter “flash bootloader”
