  • 7.6.1.2 SYR_DBL15: execute debug/diagnosis support (XCP/INCA) - obsolete
  • 7.6.1.2.1 Description
  • 7.6.1.2.1.0-1 Brief description:
  • 7.6.1.2.1.0-1.0-1 This UC contains the communication on XCP-Protocol level: The data to be transferred/received will be embedded into XCP-valid datagrams. The handling of the network-layer and below is not scope of this UC. On demand this UC will take received data from outside of CCU (e.g. via CAN/Ethernet) and process requested action. This UC will generate the XCP timestamps on demand. This UC will combine values to blocks on demand. This UC will acknowledge external request autonomously (response) This UC will pass events from CCU to external device (asynchronous event) This UC will process service requests (service request) This UC will process CYCLIC data acquisition on request by external tool (periodic measured values) This UC will stimulate the CCU on demand (external request) (Stimulation) This UC will process the security access request from external device and will grant access to protected data if authentication passed successfully. It’s expected this this UC will be implemented by BAC-Software completely. Only configuration has to be done by Panasonic.
