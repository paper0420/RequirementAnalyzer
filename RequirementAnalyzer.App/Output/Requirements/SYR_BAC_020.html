  • 7.9.1.3 SYR_BAC_020: SnapS Record
  • 7.9.1.3.1 Description
  • 7.9.1.3.1.0-1 Brief description: The Snaps Collects the ECU Internaldata for status monitoring. The Snaps Data can be accessible ouside the ECU via Diagnositic channels. The Snapshot Recorder allows the sampling of data in a ring buffer, to analyze the development of certain data sources over time.
  • 7.9.1.3.1.0-2 Preconditions: - CCU shall be in running state - Secoc Bypass shall be OFF - Coding parameters shall be enabled(SNAPSHOT_RECORDER_ENABLED). - Secure data and Time Message Pdus shall be recieved. - SFA Feature shall be enabled onlz when the Post Build configuration. Currently In the CCU Post build configuration is not enabled.
  • 7.9.1.3.1.0-3 Trigger: Trigger for SnapShort creation is simple detection of a DTC.
  • 7.9.1.3.1.0-4 Input data: Zedis document Coding FA File CAF File BAC Module(SnapS)
  • 7.9.1.3.1.0-5 Description of behaviour: Snapshot recorder uses various types of data sources as input. These data sources are consistes of single value or sampled ring buffer structures. These data will be stored in chronological way. Snapsrecorder data shall be polled with a periodic runnable. Snapshort recorder also supports compression algorthims for data values. Below are the trigger conditions for triggering the snaps recorder. 1) Data trigger 2) Call triggers 3) Diagnostic event trigger Below are the diagnostic routines need to support 1) SR_DELETE_ALL_SNAPSHOTS (Service 0x31 - RoutineControl, RID: 0x101B) 2) STATUS_SR_CONTROL_INFO (Service 0x22 - ReadDatabyIdentifier, DID: 0x1510) 3) STATUS_SR_AVAILABLE_SNAPSHOTS (Service 0x22 - ReadDatabyIdentifier, DID: 0x1511) 4) SR_SNAPS_ADUE (Service 0x31 - RoutineControl, RID: 0x101E) Additional Memory Information for Snaps : 1) Number of snapshorts required for CCU is 3. 2) Number of Datasets required for CCU is 15 and Each Dataset Size is 100 bytes & ring buffer size is 7. 3) Size of the memorypoll is 10240 bytes.
  • 7.9.1.3.1.0-6 Timing Requirements: Time gap between 2 snapshort creation request requires 30 sec.
  • 7.9.1.3.1.0-7 Output data: Snapshort data creation
  • 7.9.1.3.1.0-8 Postconditions: none
  • 7.9.1.3.1.0-9 Descriptions and exceptions: none
  • 7.9.1.3.1.0-10 Dependencies and interactions: Below are the interacted modules : 1) Coding 2) DEM 3) DCM 4) Snaps 5) NVM 6) BS 7) CSM 8) Cryif 9) Crypto
