  • 7.5.1.38 SYR_DIA_370: prevent SW-Update if PWF is not in PAD-Mode
  • 7.5.1.38.1 Description
  • 7.5.1.38.1.0-1 Brief description: on request of session-change to "Update", the CCU shall check the PWF-Mode. If PWF-Mode is NOT "PAD", the session change shall be rejected (=condition not correct) If PWR-Mode is NOT available (by missing CAN-Message), SW-Update shal lbe possible
  • 7.5.1.38.1.0-2 Preconditions: ESYS-Connected to CCU and request "session change to Download"
  • 7.5.1.38.1.0-3 Trigger: PWF-State and session-request by UDS
  • 7.5.1.38.1.0-4 Input data: none
  • 7.5.1.38.1.0-5 Description of behaviour: CCU will check PWF-state on request "session change to download", if PAD mode is not set, CCU will send "negative response"
