  • 7.7.1.17.1 Description
  • 7.7.1.17.1.0-1 Brief description: Secure boot process
  • 7.7.1.17.1.0-2 Preconditions: none
  • 7.7.1.17.1.0-3 Trigger: CCU boot trigger
  • 7.7.1.17.1.0-4 Input data: SW flashed binaries
  • 7.7.1.17.1.0-5 Description of behaviour: HSM Start up - The Bootmanager memory region shall be designated as OTP. - The Bootmanager shall be the sole entry point of booting sequence. - Upon it's startup, the Bootmanager shall initiate the HSM's startup sequence till release into the HSM state "CORE_APP" - The Bootmanager shall configure the HSM's clock before configuring the Core6M's. - The HSM shall be configuered to the maximum clock, by using a clock divider of 1 in ecy_HSM_ClockConfigure CSAI. - Logic Expression "ConfigurationUpdateCheck" definition is: if ( TB_ConfigurationUpdate AND !(SFA_PermAuth_IsEnabled) ) - When the "ConfigurationUpdateCheck" is evaluated as FALSE, the bmgr shall not allow any changes to the Trusted Boot Reference Table. - When the "ConfigurationUpdateCheck" is evaluated as TRUE, the bmgr shall: Apply requested change as in CCU-11040 - Logic Expression "ValidApplicationCheck" definition is: if ( C1==False || C2==True ) - When the "ValidApplicationCheck" is evaluated as TRUE, the bmgr shall switch to btld, following the sequence in section: BootSequence Usecase: Boot to Bootloader . - When the "ValidApplicationCheck" is evaluated as FALSE, the bmgr shall switch to Appl, following the sequence in section: BootSequence Usecase: Boot to Application BootSequence Usecase: Boot to Bootloader - Logic Expression "AuthenticatedBootCheck" definition is: if ( TB_IsBtldValid || !(SFA_PermAuth_IsEnabled || SFA_TempAuth_IsEnabled ) ) - When the "AuthenticatedBootCheck" is evaluated as TRUE, the bmgr shall switch to btld. - When the "AuthenticatedBootCheck" is evaluated as FALSE, the bmgr shall: 1. Clear the C1/programming request flag. 2. Set "ErrorReport" flag as "BootCMAC". 3. Reset. To allow booting to the Application instead. - When the HSM is fallible, the bmgr shall: 1. Evalute "TB_IsBtldValid" = FALSE. 2. Set "ErrorReport" flag as "HSMFail". BootSequence Usecase: Boot to Application - The bmgr shall await a positive confirmation of the Disable Security Critical Fns CSAI of the HSM before resuming boot to Appl. - The bmgr shall signal the HSM that end of boot process before switching to Appl to trigger RTMD check. - The bmgr shall implement a retry mechanisim in case of negative or missing HSM CSAI resp. - If HSM error persist, the bmgr shall set the "ErrorReport" as "HSMFail" before booting to Appl.
