SYR_PLC_05


  • 1.1.30.6 SYR_PLC_05: provide software function to generate random traffic over PLC channel for PSD measurement
  • 1.1.30.6.1 Description
  • 1.1.30.6.1.0-1 Brief description: For measurement purpose, software shall generate random data HPGP traffic over PLC channel. This Function can be enable or disabled via Diag JOB. The DUT shall transmit use 20% if maximum data rate as define in ISO15118-3 [V2G-A11-3]. Size of Data 1014 bytes Speed: 1Mb/s Tx Frame Detail: Destination MAC address FF:FF:FF:FF:FF:FF (broadcast address) Source MAC address (MAC of ST Chorus) VLAN (optional) Ethertype (0x88E1) Body : 01010101....(length Max. 1014) or any random values
  • 1.1.30.6.1.0-2 Preconditions: PHY_LINK shall be established
  • 1.1.30.6.1.0-3 Trigger: diagnostic job
  • 1.1.30.6.1.0-4 Input data: NA
  • 1.1.30.6.1.0-5 Description of behaviour: LIM_C shall trigger to establish PHY_LINK. Diag JOB shall check the PHY_LINK establish before starting the traffic. When Enabled by Diag Job Tx frame should be seen at least 20% of Max. data rate When Disable, it should stop and PhyLink should be maintained.
  • 1.1.30.6.1.0-6 Timing Requirements: no special customer request
  • 1.1.30.6.1.0-7 Output data: output on PLC Bus
  • 1.1.30.6.1.0-8 Postconditions: PHY_LINK can remain establish
  • 1.1.30.6.1.0-9 Descriptions of exceptions: if PHY_LINK is not establish before starting traffic, JOB should provide error.
  • 1.1.30.6.1.0-10 Dependencies and interactions:
  • 1.1.30.6.2 Differences to CRS
  • 1.1.30.6.2.0-1 Dependencies: Diagnostic Job or XCP Lable shall trigger this funtion.
