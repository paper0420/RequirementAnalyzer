  • 7.10.1.1.168.1.0-1 Brief description:Normal application automatically satisfies this requirement hence no addtional action required in this SM Feedback from ST: In there RAM there are locations that we call All-X. For these locations, the codeword with all bit set to 0b or to 1b is a valid codeword. The ECC logic assesses the codeword to correct or to detect bit error in RAM. If the codeword is valid, there is not bit flip in the data. Typically in the RAM, these All-X locations are periodically placed in the array and there is one All-X location every 255 normal locations. The requirement asks NOT to read only All-X locations in each FTTI. But a real application - which has a stack, a heap and other data in RAM - doesn't read only All-X locations. As a consequence, a normal application automatically satisfies this requirement. You must confirm that you don't have a weird application that jumps from one All-X location to another All-X location without reading normal locations.
