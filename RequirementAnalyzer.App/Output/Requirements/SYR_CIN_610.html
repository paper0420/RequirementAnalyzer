  • 8.1.65 SYR_CIN_610 :Deactivation_Debug_PDUs_for_serie
  • 8.1.65.1 Description
  • 8.1.65.1.0-1 Brief description: The Debug PDUs are sent cyclically on the bus to help for debugging on development cars. MessIdCcuEndOfLineTest MessIdCcuEndOfLineTestNo2 MessIdCCUEndOfLineTestNo3 DebugCombinedChargerUnitAeCanFd These debug frames must be deactivated in Field Mode. The debug frames (not only in CCU) are leading to +8% bus load increase on AE_CAN_FD. Disabling these debug PDUs in Field Mode will help in reducing the Bus load.
  • 8.1.65.1.0-2 Preconditions: CCU should be in running state.
  • 8.1.65.1.0-3 Trigger: Change of modes between Field, Plant and Engineering Mode.
  • 8.1.65.1.0-4 Input data: Read the current ECU Mode from Pana_SWC_Security component
  • 8.1.65.1.0-5 Description of behaviour: If the current Mode is Field Mode, Bswm should read the current mode and accordingly the Below pdus shall be disabled. MessIdCcuEndOfLineTest MessIdCcuEndOfLineTestNo2 MessIdCCUEndOfLineTestNo3 DebugCombinedChargerUnitAeCanFd This pdu( DebugCombinedChargerUnitAeCanFd) is not applicable for G26 and other varients it is applicable(I20 and G70). If the current mode is Plant\Engg. mode then these pdus should be transmitted on AE-CANFD channel.
  • 8.1.65.1.0-6 Timing Requirements: NA
  • 8.1.65.1.0-7 Output data: NA
  • 8.1.65.1.0-8 Postconditions: These pdus will be deactivated in the Field Mode.
  • 8.1.65.1.0-9 Descriptions of exceptions: NA
  • 8.1.65.1.0-10 Dependencies and interactions: Below the Autosar Modules are interacted: --> BswM --> Com --> RTE --> Application Component(Pana_Swc_Security )
