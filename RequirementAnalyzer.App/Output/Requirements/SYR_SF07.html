  • 7.11.2.9 SYR_SF13 FuSa : Safe NVM Support
  • 7.11.2.9.1 Description
  • 7.11.2.9.1.0-1 Brief description: The CCU may need support for safe NVM storage ensuring that NVM data from FuSa relevant partitions is stored in a safe way although the infrastructure located in BSW has only QM safety level. SM: Application level redundant storage ASIL data storage to be handled in the respective application component. Use redundant copies of safety relevant data and perform coherence check before use To address missing writes on EEPROM cells, it is required that the application software executes a read-back of written data after an update of the EEPROM values. Missing writes will be handled as a hardware fault AUTOSAR memory stack(NVRAMManager->MemoryAbstractionInterface->FlashEEPROMEmulation->FlashDriver) shall remains QM Application shall store the redundant data in different memory blocks. Application shall readback and check the data after writing in to the memory Application shall also do the plausibility check each time before using the safety data.
