  • 1.1.30.7 SYR_PLC_06: provide failure detection
  • 1.1.30.7.1 Description
  • 1.1.30.7.1.0-1 Brief description: provide failure detection in establishing PLC channel for communication. When Failure is detected which cannot be recovered runtime, DTC: Powerline communication : modern error shall be rasies and enviorment variable shall be saved by CCU.
  • 1.1.30.7.1.0-2 Preconditions: Normal operation for establishing the PLC Channel
  • 1.1.30.7.1.0-3 Trigger: detection of failure
  • 1.1.30.7.1.0-4 Input data: NA
  • 1.1.30.7.1.0-5 Description of behaviour: - request for LIM_C to establish PHY Link, PLCCP shall perform the procedure to establish PHY LINK. - If the QCA chip is not powering up - If the QCA chip is not responding - If the QCA chip is not booting up (internally or externally) - If the QCA chip is not responding with correct SW version On above error we will fail to establish PHY link thie failure shall be detected and DTC 0x03194A shall be set with UWB expected. PLCCP shall report the error as PASSED/FAILED after checking all above condition to recover to DCM. DCM will store this in fault memory. If the failure is Passed for next 40 device wakeup cycle the fault memory will be removed. Debounce count shall be set to zero.
  • 1.1.30.7.1.0-6 Timing Requirements: NA
  • 1.1.30.7.1.0-7 Output data: DTC shall be raised when failure is detected
  • 1.1.30.7.1.0-8 Postconditions: DTC and enviorment data can be retreivable via diagnostic tester
  • 1.1.30.7.1.0-9 Descriptions of exceptions: NA
  • 1.1.30.7.1.0-10 Dependencies and interactions: PLCCP state machine
