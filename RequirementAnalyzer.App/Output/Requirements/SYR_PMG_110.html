SYR_PMG_110


  • 7.2.1.9 SYR_PMG_110: execute CAN-Powermanagement
  • 7.2.1.9.1 Description
  • 7.2.1.9.1.0-1 Brief description:
  • 7.2.1.9.1.0-2 This UC is NOT valid for the CHAdeMO-CAN! The CCU will never wake up any connected CAN. The CAN-Powermanagement is mainly controlled by AUTOSAR-CAN-NM, PnS (Partial Network Slave) and CanSM (Autosar CAN statemanager). The complete configuration of wakeup-reasons and reasons to permit/deny CAN-Bus-Sleep is supplied by BMW. Panasonic will just integrate Autosar-BSW and supplied BAC (BMW Autosar Core)
  • 7.2.1.9.1.0-3 Preconditions: none
  • 7.2.1.9.1.0-4 Trigger: none, since it's permanent active, even the CAN is not running
  • 7.2.1.9.1.0-5 Input data: CAN-Messages , V2G-Signalling, internal states (RTE)
  • 7.2.1.9.1.0-6 Description of behaviour: depending on input (level, event) the designated state of CAN-Bus will be selected
  • 7.2.1.9.1.0-7 Timing Requirements: no direct on, the Autosar-NM requests timing constrains, which are not considered here
  • 7.2.1.9.1.0-8 Output data: designated CAN-operation Mode (NORMAL, Prepare-Sleep, Sleep, Wakeup(RM))
  • 7.2.1.9.1.0-9 Postconditions: none
  • 7.2.1.9.1.0-10 Descriptions of exceptions: none
  • 7.2.1.9.1.0-11 Dependencies and interactions: none
  • 7.2.1.9.2 Differences to CRS
  • 7.2.1.9.2.0-1 none
  • 7.2.1.9.3 Questions and Answers
