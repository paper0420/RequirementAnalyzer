  • 7.9.1.16 SYR_COA_090: Support partial networking via BSWM rules
  • 7.9.1.16.1 Description
  • 7.9.1.16.1.0-1 Brief description: This UC gives brief description of the COM Adapter determining partial networking COM Adapter shal detect all the signals that are not available in the current subnet by reading the mode switch notifications from BSWM and comparing with the message catalogue information. During the Standby, Ready, Active modes the system bus CAN communication is active. In these states the system state Park, Dwell, Drive is communicated and mapped to sub states. Com Adapter updates suitable value in the internal qualifier depending on the subnetwork status. The BswM shall receive the current COM communication status as active or not. For example the Network management message NM3 received from sub net master "CON_VEH" is used to by COM adapter to get the partial network information and also gets this information from mode switch notifications. When the NM3 message is active for particular partial network then the the partial network status is set as ON and when the NM3 message is inactive for particular partial network then the the partial network status is set as OFF. COM Adapter shall consider the message catalogue and Xml schema information to create IPDU groups and setting the qualifiers to the signals.
