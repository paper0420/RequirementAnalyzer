SYR_CCS_604


  • 1.1.31.18 SYR_CCS_604: Interfaces re-naming for Vector common callbacks between PANA SCC adaptor and BMW SWC (CIF) for ISO DC-EIM and DIN.
  • 1.1.31.18.1 Description
  • 1.1.31.18.1.0-1 Brief description: Some RTE interfaces are using same callbacks for DIN and ISO DC-EIM and can be used as common interface. 1. xxxCcsDIN signals will be renamed to xxxCcs (same interface with new name). 2. xxxCcsISO interfaces will be deleted.
  • 1.1.31.18.1.0-2 Preconditions:Flap opened. Vehcile is connected to Charger. Proximity detection.
  • 1.1.31.18.1.0-3 Trigger:EVSE applies valid duty-cycle(5%) on Control Pilot-Line and LIM-C initiates the communication
  • 1.1.31.18.1.0-4 Input data:analog values of Pilot-Line (duty-cycle/voltage level), proximity-line voltage level, PLC-communication of Pilot-Line.
  • 1.1.31.18.1.0-5 Description of behaviour: In G70, CCU supports ISO15118 DC-EIM charging together with existing DC charging as per DIN standard (from G08 variant). Current situation in G70: Vector SIP has some common callbacks for ISO DC-EIM and DIN functions. CCU SW is using existing DIN interfaces while invoking common callbacks (which are 29 in total including Rx and Tx Call Backs both.) during ISO DCEIM communication. New Situation with CR: BMW decided to create new interfaces (generic interface names) common for DIN and ISO DCEIM both. Once the new interfaces are available from BMW in arxml, Supplier SWC shall make connections to these new interfaces ports and invoke corresponding common vector callbacks in both DIN and ISO DCEIM. Changes will be at RTE read/write calls, Port connections & naming, SCC adaptor module. New Interface names and corresponding callbacks are captured in LIM HW excel sheet: https://ccu-svn.microfuzzy.com/CCU_G26/SP21/SP21_SOP_2207/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/interface Sequence flow for ISO DCEIM and DIN DC valid for G70 are at: https://ccu-svn.microfuzzy.com/CCU_G26/SP21/SP21_SOP_2207/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/Sequence
  • 1.1.31.18.1.0-6 Timing Requirements: define by ISO15118-2 & DIN70121
  • 1.1.31.18.1.0-7 Output data: RTE interfaces (Rx or Tx) shall be updated correctly while charging communication.
  • 1.1.31.18.1.0-8 Postconditions: DIN70121 and ISO15118 DC-EIM charging communication shall be established between EV and EVSE.
  • 1.1.31.18.1.0-9 Descriptions of exceptions:NA
  • 1.1.31.18.1.0-10 Dependencies and interactions:BSW (SCC), PANA Appl. (SCC adaptor), BMW SWC, RTE, Arxml
