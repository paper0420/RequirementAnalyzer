  • 7.2.1.33 SYR_PMG_360: Locking Wakeup via pilot in case of a fully charged car
  • 7.2.1.33.1 Description
  • 7.2.1.33.1.0-1 Brief description: This implementation is to selectively enable / disable wakeup via Control Pilot line (CP) and AC/DC Proximity line based on the value on the BMW interface BMW_st_PiwProxWkuAll
  • 7.2.1.33.1.0-2 Preconditions: Read the value of the interface BMW_st_PiwProxWkuAll and then trigger CCU sleep.
  • 7.2.1.33.1.0-3 Trigger: CCU is in sleep state and trigger wakeup via CP or Proximity line.
  • 7.2.1.33.1.0-4 Input data: none
  • 7.2.1.33.1.0-5 Description of behaviour: There is no possibility of selectively enabling / disabling Wakeup via Control Pilot line (CP) and Proximity line based on the value on the BMW interface BMW_st_PiwProxWkuAll . The current implementation enables wakeup via both CP and Proximity line if the value of BMW_st_PiwProxWkuAll != 2 (Disabled) and disables the wakeup when BMW_st_PiwProxWkuAll = 2. In case of a fully charged vehicle the wakeup by pilot should be blocked. For this implementation, the existing interface (BMW_st_PiwProxWkuAll ) will be used as defined below: 0000 0000b – Wakeup Proximity and Pilot enabled (Bit0) 0000 0001b – Wakeup Pilot disabled  (Bit0) 0000 0010b - Wakeup Proximity disabled (Bit1) 0000 0011b - Wakeup Proximity and Pilot disabled (Bit0+1)
