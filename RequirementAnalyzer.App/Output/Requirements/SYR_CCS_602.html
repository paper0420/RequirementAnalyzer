  • 1.1.31.16 SYR_CCS_602 : Provide CCS Protocol Support for ISO 15118 DC EIM Message Set
  • 1.1.31.16.1 Description
  • 1.1.31.16.1.0-1 Brief description: Supplier shall provide CCU software which supports ISO15118(edition 1) DC EIM message. Vector stack bring-up & configurations shall be done by supplier to enable DC EIM between CCU and EVSE (charging station). Within CCU Software , PANA SW (BSW & Appl. SW) communicates with BMW SWC (example, CIF) over RTE interface.
  • 1.1.31.16.1.0-2 Preconditions: Flap opened. Vehcile is connected to Charger. Proximity detection.
  • 1.1.31.16.1.0-3 Trigger: EVSE applies valid duty-cycle(5%) on Control Pilot-Line and LIM-C initiates the communication
  • 1.1.31.16.1.0-4 Input data: Analog values of Pilot-Line (duty-cycle/voltage level), proximity-line voltage level, PLC-communication of Pilot-Line.
  • 1.1.31.16.1.0-5 Description of behaviour: CCU will communicate with EVSE like described in ISO-15118(edition-1). ISO DC Container bring-up & configuration includes EXI streams/structs size, configuration parameters (timeouts), read/write to NVM blocks and callback parameters. Please refer seq (applicable for G70) https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/Sequence/LH_CCS_ISO15118_DC_EIM_Sequence.pdf Supplier software shall use Get / Set callbacks in Vector SW. With "Get" callback, message parameter value is received from BMW SW (via RTE interface) and send it to EVSE in request during charging session. With "Set" callback, message parameter value received in response from EVSE is send to BMW SW via RTE interface. Please refer the below link for DC EIM message parameters and associated callbacks & interfaces. https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/interface/ISO_Flow_with_CallBacks.xlsx https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Charging_Interface/CCS/interface/LIM-HW_Information_v1.xlsx Agreement with BMW: Charging Profile is not calculated on Vehicle side as there is no SOME/IP communication for DC EIM feature. Supplier software will take a default SAScheduletuple ID (first entry) and send it to EVSE in Power Delivery Request. No Charging Profile data in Power Delivery.
  • 1.1.31.16.1.0-6 Timing Requirements: define by ISO15118 (edition-1)
  • 1.1.31.16.1.0-7 Output data: defined by ISO15118(edition 1)
  • 1.1.31.16.1.0-8 Postconditions: DC EIM based charging session is established between EV and EVSE.

  • 1.1.31.16.1.0-10 Dependencies and interactions: BSW (SCC, PLC, ETH, SPI), PANA Appl., BMW SWC, RTE
