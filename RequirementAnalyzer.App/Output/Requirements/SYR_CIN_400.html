  • 8.1.44 SYR_CIN_400: Handling of EFAN Meta-Data Query
  • 8.1.44.1 Description
  • 8.1.44.1.0-1 Brief description: Request is to provide BMW part number recieved from new EFAN to BMW application interface. Additionally, a new meta-data query shall be executed to fetch EFAN specific data which shall be provided to application interface. The decision of old EFAN or new EFAN will be controlled by BMW interface. Based on the decision, supplier SW shall execute required Master requests-resp over LIN and map data accordingly.
  • 8.1.44.1.0-2 Preconditions: CCU is running
  • 8.1.44.1.0-3 Trigger: LIN wake-up
  • 8.1.44.1.0-4 Input data: Concept document at SVN shall be referred: https://svn-bmw03-pais.eu.panasonic.com/BMW03_CCU_DOC/Common/WorkSpace/ENG.02_SystemRequirementsAnalysis/ENG.02_SystemRequirementSpecification/AdditionalDocuments/Documentation/Thermal_Mgmt/CCU-25574_LIN_EFAN_Meta-data_Query_Latest.pptx
  • 8.1.44.1.0-5 Description of behaviour: EFAN port in BMW application is changed from BMW_rc_DataFan_rc to BMW_rc_DataFan1_rc. Supplier shall map the SlaveDiagResp data to this new port name (signal elements). New BMW interface i.e. BMW_b_FanDataTypOld controls whether it is old EFAN or new EFAN. BMW shall set the value of interface in init runnables and shall provide in 200ms. Supplier SW shall react on interface value. If BMW_b_FanDataTypOld is set to true, supplier shall consider Old EFAN as Thermal Slave. 1. CCU (LIN master) shall trigger Master request with ID 0x29, EFAN specific data is expected to be received from slave and the supplier shall provide resp. to BMW interface and sensor identification lesen job response data. Slave Resp. shall map to same signal elements as defined in current SW. Refer concept document. 2. Old EFAN does not provide BMW part number. Therefore, supplier shall set BMW_rc_DataFan1_rc.BMW_st_FanCptNr = 0 If BMW_b_FanDataTypOld is set to false, supplier shall consider new EFAN as Thermal Slave. 1. CCU (LIN master) shall trigger new Master request with identifier 0x32, new EFAN slave shall provide E-FAN specific data as response. Supplier shall map the response data to corresponding signal elements (on port BMW_rc_DataFan1_rc) which is same as elements mapping done for old EFAN specific data (with ID 0x29). 2. New EFAN specific data (resp. of req. ID 0x32) is not provided to sensor identification lesen job. 3. CCU (LIN master) shall trigger Master request with ID 0x29, BMW part number is expected to be received from slave and the supplier shall provide to BMW interface and sensor identification lesen job response. Resp. data shall map to new signal element i.e. BMW_rc_DataFan1_rc.BMW_st_FanCptNr. BMW part number shall be provided in HEX format. Refer concept document for mapping of bytes received in response to element.
