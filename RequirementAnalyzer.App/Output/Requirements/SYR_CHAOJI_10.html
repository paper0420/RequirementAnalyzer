SYR_CHAOJI_10


  • 1.1.28.2 SYR_CHAOJI_10 :Activation GBT communication with Chaoji coding configuration
  • 1.1.28.2.1 Description
  • 1.1.28.2.1.0-1 Brief description: The Chademo discreet signal hardware and CAN (EAPC_DC CAN) hardware is shared with the DC-China. The configuration need to be adapted based on coding entries. The Coding variable will be decide based on Car type and country. The coding table is expected by customer. BMWcod_st_VehChgrTyp DC coding parameter indicating the DC charging type can have updated value ‘ 4’ GBT protocol shall be activated . The CAN Baudrate shall be configurad to 250 Kbps. Please refer 'SYR_GBTDC_11 & SYR_GBTDC_10 ' for more details of GBT protocol communication.
  • 1.1.28.2.1.0-2 Preconditions: NA
  • 1.1.28.2.1.0-3 Trigger: coding paramter BMWcod_st_VehChgrTyp has value 4
  • 1.1.28.2.1.0-4 Input data: CAF file
  • 1.1.28.2.1.0-5 Description of behaviour: BMWcod_st_VehChgrTyp DC coding parameter with value 3 indicates Dc charging type is GBT & BMWcod_st_VehChgrTyp DC coding parameter with value 4 indicates Dc charging type is ChaoJi . EAPC_DC can will communicate according to GBT protocol . and the Baudrate of EAPC_DC can will be 250kbps to support the GBT standard
  • 1.1.28.2.1.0-6 Timing Requirements: NA
  • 1.1.28.2.1.0-7 Output data: NA
  • 1.1.28.2.1.0-8 Postconditions: Baudrate on EAPC_DC can is changed to 250kbps
  • 1.1.28.2.1.0-9 Descriptions of exceptions: NA
