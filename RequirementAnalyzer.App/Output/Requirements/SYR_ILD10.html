SYR_ILD10


  • 1.1.2.1 SYR_ILD10: CCU interface with Intelligent plug via Private CAN bus
  • 1.1.2.1.1 Description
  • 1.1.2.1.1.0-1 Brief description: The communication interface between CCU and intelligent plug is realised through Private CAN bus. 500 Kbps baud rate and 11 bit identifier shall be used. The message catalogue shall contain the following address ranges. 0x01 - 0x0F ---- State Control 0x10 - 0x2FF ---- Sensors and actuators 0x300 - 0x5FF ---- Cyclic component status 0x600 - 0x6FF ---- XCP messages 0x700 - 0x7FE ---- Diagnosis communication For more details refer to LH specification LH 10000452-000 - 12 and LH_Private-CAN_BL1.1 (LH 11028788 )
  • 1.1.2.1.1.0-2 Preconditions:
  • 1.1.2.1.1.0-3 Trigger: Private CAN communication is active when CCU is active
  • 1.1.2.1.1.0-4 Input data: ST_CON_VEH = 1000b
  • 1.1.2.1.1.0-5 Description of behaviour: The private CAN bus shall handle the following signals. - A_T_LED_G : Charge status LED Green - A_T_LED_R : Charge status LED Red - M_LED_W : LED searching light GND - A_T_LED_W : Search LED supply - LED_GND : Charge status LED GND - A_T_LED_B : Charge status LED Blue - E_A_TEMP_DC1 : DC charging socket temperature sensor #1 - E_A_TEMP_DC2 : DC charging socket temperature sensor #2 - M_FLAP_LOCK : 1st Flap lock Ground - E_A_FLAP_LOCK : 1st Flap lock position - E_S_PLUG_LOCK_SIGNAL : Pluck lock (used as flap lock for CHAdeMO) - A_T_PLUG_LOCK+ : Pluck lock (used as flap lock for CHAdeMO) - E_A_TEMP_AC2 : EVSE socket temperature sensor #2 - A_U_CUR_SENS_DC : DC plug current sensor vcc - M_DC_SENS : DC plug current sensor ground - E_A_CUR_SENS_DC : DC plug current sensor input - A_T_FLAP_LOCK+ : 1st Flap lock + - A_T_FLAP_LOCK- : 1st Flap lock - - A_T_PLUG_LOCK- : Pluck lock (used as flap lock for CHAdeMO)
  • 1.1.2.1.1.0-6 Timing Requirements:
  • 1.1.2.1.1.0-7 Output data: All the Private CAN signals will be updated with current data values.
  • 1.1.2.1.1.0-8 Postconditions: Vehicle is in "Prepare to drive" position
