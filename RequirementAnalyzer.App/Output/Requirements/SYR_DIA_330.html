SYR_DIA_330


  • 7.5.1.34 SYR_DIA_330: measure and read CPU Load
  • 7.5.1.34.1 Description
  • 7.5.1.34.1.0-1 Brief description: The maximum average CPU load is continously logged into NVM. Update in NVM occurs only if the average CPU load coming from GlIWA measurement is exceeded. See EPIC description for further details.
  • 7.5.1.34.1.0-2 Preconditions: Software runs
  • 7.5.1.34.1.0-3 Trigger: A new value will be generated approx. every 2 sec.
  • 7.5.1.34.1.0-4 Input data: GLIWA kernel measurement value
  • 7.5.1.34.1.0-5 Description of behaviour: This is a cyclic task.
  • 7.5.1.34.1.0-6 Timing Requirements: This is a low priority task with no real-time constraints. Realtime requirements are satisfied by the GLIWA measurement kernel.
  • 7.5.1.34.1.0-7 Output data: Current maximum average load for each core
  • 7.5.1.34.1.0-8 Postconditions: NVM remains over resets and is erased when the CCU is flashed.
  • 7.5.1.34.1.0-9 Descriptions of exceptions: none
  • 7.5.1.34.1.0-10 Dependencies and interactions: none
  • 7.5.1.34.2 Differences to CRS
  • 7.5.1.34.2.0-1 CRS mentions maximum runtime measurements. However, this value does not provide a reasonable amount of system observation because the actual runtime may vary due to system jitter. Update frequency is reduced from orignally requested 20 ms to the 2 sec above.
