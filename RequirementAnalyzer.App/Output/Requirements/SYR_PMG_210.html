  • 7.2.1.20 SYR_PMG_210: startup device
  • 7.2.1.20.1 Description
  • 7.2.1.20.1.0-1 Brief description:
  • 7.2.1.20.1.0-2 This UC gathers all actions, needed to execute a start up of CCU. It reflects the “startup path” of SYR_PMG140. While start up it’s checked if all components are started correctly before depending components are started. In case of startup-delay by some component, this UC will monitor this component and decide which impact is caused by this fail. In worst case the UC will decide to drop certain CCU functionality caused by failed start of a component. This UC will generate log entries in persistent log-buffer for offline analysis of problems
  • 7.2.1.20.1.0-3 Preconditions: CCU is in power-down mode
  • 7.2.1.20.1.0-4 Trigger: any wake up trigger
  • 7.2.1.20.1.0-5 Input data: wakeup reason
  • 7.2.1.20.1.0-6 Description of behaviour: CCU will startup to handle wakeup event
  • 7.2.1.20.1.0-7 Timing Requirements: <150ms CAN communication (RX) must be possible
