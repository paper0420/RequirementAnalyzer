SYR_PMG_330


  • 7.2.1.30 SYR_PMG_330: Disable Wakeup during unhealable charging error
  • 7.2.1.30.1 Description
  • 7.2.1.30.1.0-1 Brief description: The CCU wakes up due to the missing PE-contact which is the ground line for the CP and Proximimity measurement. When PE is not there, the CP signal and the proximity signal are floating, which causes an unexpected wakeup via CP or Proximity change. The problem is severe in the consequences , To solve the problem, a robustness countermeasure is to be implemented by which Wakeup Triggers via Proximity or Pilot line can be locked/disabled by BMW SW. Ethernet WUP should not be affected.
  • 7.2.1.30.1.0-2 Preconditions: CCU is operational.
  • 7.2.1.30.1.0-3 Trigger: Removal of the PE contract
  • 7.2.1.30.1.0-4 Input data: none
  • 7.2.1.30.1.0-5 Description of behaviour: An Interface from BMWHVcha BMW_st_PiwProxWkuAll will be used to disable/enable the wakeup. BMW_st_PiwProxWkuAll: 0 – Function not initialized 1 – Wakeup allowed 2 – Wakeup disabled
