SYR_SEC_240


  • 7.7.1.23 SYR_SEC_240: SZ Performance Enhancement G26 BEV
  • 7.7.1.23.1 Description
  • 7.7.1.23.1.0-1 Brief description: A signed SFA- request will be generated by BMW, this will be transfered via diagnosis to CCU. CCU will check signature, in case of valid signatiure, Once the signature verification is valid the SFA feature will activate. Accordingly Performance mode will be activiated or disabled.
  • 7.7.1.23.1.0-2 Preconditions: SFA feature (0xAA0170) shall be activiated and corresponding CAF file shall be flashed.
  • 7.7.1.23.1.0-3 Trigger: SFA request by Diagnosis-Tester
  • 7.7.1.23.1.0-4 Input data: CAF file
  • 7.7.1.23.1.0-5 Description of behaviour: BMW wants to enhance the characteristic mapping for performance using a software update. BMW reads the SFA feature status and accordingly calibration data will be mapped by the BMW high level components. Double performance feature will be activated once SFA feature id is enabled. Normal Mode: In case SFA feature is not activated, Normal mode will be used.
  • 7.7.1.23.1.0-6 Timing Requirements: none
  • 7.7.1.23.1.0-7 Output data: NA
  • 7.7.1.23.1.0-8 Postconditions: NA
  • 7.7.1.23.1.0-9 Descriptions of exceptions: none
  • 7.7.1.23.1.0-10 Dependencies and interactions: BMW BAC Module (BS) BMW software component (BMW_SWC_PciIn) and software composition (BMW_CPS_PCI) connection
