#SYQT-68-0001-0009#

[Charging],[DC_Charging],[Proximity]
Test Objective: Checking CHAdeMO's charging operation sequence (input / output)

Requirements

  • 7897 -
    [Not Changed][accepted][HWT (ENG.7/8)Review][Requirement]
    VARIANT (DC Charging): The "CCU" must implement the CAN communication with the charging station according to CHAdeMO 1.0.1 chapter 6 "Circuit requirements".
  • 19049 -
    [Not Changed][partly accepted][SWQT (ENG.8)][Requirement]
    The CHAdeMO DC charging procedure shall be implemented.
  • 19060 -
    [Not Changed][accepted][SWQT (ENG.8)][Requirement]
    Every 10 ms, the supplier function shall send the current state of the Charge_Sequence_1 line ("E_HV+_RELAY") to the "Ladeschnittstelle" function via the "Status_Charge_Sequence_Signal_1" interface signal (0 = inactive, 1 = active, 2 = error)
  • 19061 -
    [Not Changed][accepted][SWQT (ENG.8)][Requirement]
    Every 10 ms, the supplier function shall send the current state of the Charge_Sequence_2 line ("E_HV-_RELAY") to the "Ladeschnittstelle" function via the "Status_Charge_Sequence_Signal_2" interface signal (0 = inactive, 1 = active, 2 = error)
  • 19083 -
    [Not Changed][accepted][SWQT (ENG.8)][Requirement]
    The supplier function shall be able to receive messages from the CHAdeMO charger including two with the identifiers 0x108 (Chademo_H108) and 0x109 (Chademo_H109).
  • 19092 -
    [Not Changed][accepted][SWQT (ENG.8)][Requirement]
    The supplier function shall be capable of sending the CHAdeMO charger three messages with the identifiers 0x100 (Chademo_H100), 0x101 (Chademo_H101) and 0x102 (Chademo_H102) (see table: Overview of DC-CAN signals to be sent).
